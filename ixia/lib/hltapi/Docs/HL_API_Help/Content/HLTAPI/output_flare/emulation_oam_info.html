DESCRIPTION

IxNetwork, IxOS/IxNetwork-FT

This procedure will get various data from E-OAM state machines.

SYNOPSIS - ::<namespace>::emulation_oam_info

-mode

Supported platforms Details

IxNetwork [M]

DESCRIPTION

Define the desired mode for info return.

Valid options are:

aggregate

get aggregate statistics. Depending on -action they will be topology stats or message_stats.

session

get per sesssion statistics.

DEFAULT

 None

-action

Supported platforms Details

IxNetwork

DESCRIPTION

Specify what type of data to poll.

Valid options are:

Not defined

DEFAULT

get_message_stats

-port_handle

Supported platforms Details

IxNetwork

DESCRIPTION

Statistics can be polled only from one port with one procedure call. Specify the port to poll the data from. If -mode is session, all bridges from the port will be polled.

DEFAULT

 None

-handle

Supported platforms Details

IxNetwork

DESCRIPTION

Use OAM topology handles returned by emulation_oam_config_topology. When -mode is session, and action is get_topology_stats, all topology statistics will be prepended with the topology handle.Use OAM message handles returned by emulation_oam_config_mesg when -mode is session and -action is 'get_topology_stats' to poll message statistics per message handle (message handles can be returned by emulation_oam_config_msg per_group or per_message). If a message handle is used, other handles are ignored (port_handle). A list of message handles can be polled with one call. Per session message statistic keys will be prepended with the handle that they belong to.

DEFAULT

 None

RETURN VALUES

Supported Platform Details

IxNetwork

status

$::SUCCESS | $::FAILURE

log

When status is failure, contains more information.

handle

The name of the array to be used for reading the stats. This key is returned when more than 2000 statistics values need to be retrieved. The indices of this array are the keys that you'll find explained below, and the values are the statistics values. This method has been introduced to increase performance when returning and reading stats. Returning a huge list in the tcl console can make the console hang.
