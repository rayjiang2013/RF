-mode

Supported platforms Details

IxNetwork [M]

DESCRIPTION

Deterines whether statistic filters are set or if statistic are gathered and returned in keyed lists. This following parameters are supported:

Valid options are:

set_filters

Configure statistic filters for bridge specified with -handle parameter.

get_ccm

Return PBB-TE CCM information keys for bridge specified with -handle parameter.

get_periodic_oam_lt

Return PBB-TE periodic OAM link trace learned information keys for bridge specified with -handle parameter.

get_periodic_oam_lb

Return PBB-TE periodic OAM loopback learned information keys for bridge specified with -handle parameter.

get_periodic_oam_dm

Return PBB-TE periodic OAM delay measurement learned information keys for bridge specified with -handle parameter.

get_lt

Return PBB-TE link trace learned information keys for bridge specified with -handle parameter.

get_lb

Return PBB-TE loopback learned information keys for bridge specified with -handle parameter.

get_dm

Return PBB-TE delay measurement learned information keys for bridge specified with -handle parameter.

get_all

Return all keys available.

DEFAULT

 None

-handle

Supported platforms Details

IxNetwork

DESCRIPTION

PBB Bridge handle. Statistic filters will be set on this bridge handle if -mode is set_filters. Statistic keys will be returned from this bridge handle for all other modes. This parameter is supported using the following APIs: IxN.

DEFAULT

 None

This key, handle, represents the name of the array to be used for reading the stats. This key is returned when more than 2000 statistics values need to be retrieved.
