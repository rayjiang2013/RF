NAME - ::<namespace>:: topology_config

DESCRIPTION

IxNetwork-NGPF

This command will create a topology and one or multiple device groups. This is the building block of creating a new scenario in IxNetwork.

ARGUMENTS

[M] indicates if the argument is mandatory on the supported platform.

-mode

Supported platforms Details

IxNetwork-NGPF

DESCRIPTION

Action to be taken on the interface selected. This option takes a list of values when -port_handle is a list of port handles. This option is valid for the old and the new APIs. When ::ixia::interface_config is provided with -port_handle parameter, -mode modify and other supported parameters, except -interface_handle, the modification is supported for L1 parameters only. When ::ixia::interface_config is provided with -port_handle parameter, -mode modify and other supported parameters, including -interface_handle, the modification is supported for L2-L3 parameters also, but only for protocol interfaces (-l23_config_type protocol_interface). This argument does not support lists.

Valid options are:

config

modify

Valid only when using IxTclNetwork.

destroy

Clear all interfaces before configuring new ones.

DEFAULT

config

-topology_name

Supported platforms Details

IxNetwork-NGPF

DESCRIPTION

The name of the topology that will be shown in the IxNetwork GUI. You can create at most one topology with a topology_config command. This argument is optional and does not suuport lists.

DEFAULT

 None

-topology_handle

Supported platforms Details

IxNetwork-NGPF

DESCRIPTION

Handle for the topology that the user wants to modify or delete. This argument does not support lists.

DEFAULT

 None

-port_handle

Supported platforms Details

IxNetwork-NGPF

DESCRIPTION

List of ports to be added to the topology.

DEFAULT

 None

-device_group_name

Supported platforms Details

IxNetwork-NGPF

DESCRIPTION

The name of the device group that will be shown in the IxNetwork GUI. This argument is optional and does not support lists.

DEFAULT

 None

-device_group_multiplier

Supported platforms Details

IxNetwork-NGPF

DESCRIPTION

The number of devices that will be simulated for each port assigned to the parent topology or each device simulated in the parent device group. This argument does not support lists.

DEFAULT

10

-device_group_enabled

Supported platforms Details

IxNetwork-NGPF

DESCRIPTION

This argument can be used to disable or enable individual devices within the specified device group. This argument does not support lists.

DEFAULT

 None

-device_group_handle

Supported platforms Details

IxNetwork-NGPF

DESCRIPTION

Handle for the device group that the user wants to modify, delete or use as a parent for a new device group. This argument does not support lists.

DEFAULT

 None

RETURN VALUES

Supported Platform Details

IxNetwork-NGPF

topology_handle

A list containing the topology protocol stack handles that were added by the command (if any).

device_group_handle

A list containing the device group protocol stack handles that were added by the command (if any).

EXAMPLES

SAMPLE INPUT

SAMPLE OUTPUT

NOTES

IxNetwork-NGPF

When -handle is provided with the /globals value the arguments that configure global protocol setting accept both multivalue handles and simple values. When -handle is provided with a a protocol stack handle or a protocol session handle, the arguments that configure global settings will only accept simple values. In this situation, these arguments will configure only the settings of the parent device group or the ports associated with the parent topology.
