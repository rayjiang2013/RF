-count

Supported platforms Details

IxNetwork, IxOS/IxNetwork-FT

DESCRIPTION

Number of objects to create. Only for mode create.

DEFAULT

1

-handle

Supported platforms Details

IxNetwork, IxOS/IxNetwork-FT

DESCRIPTION

Use when -mode is modify, enable or disable to specify the handles that will be altered.On mode modify only one object must be passed. On modes enable|disable|remove a list of handles can be used. Objects of type variable_descriptors cannot be enabled/disabled.

DEFAULT

 None

-mode

Supported platforms Details

IxNetwork [M] , IxOS/IxNetwork-FT [M]

DESCRIPTION

Action to perform on the object passed with -handle.create - create new object based on the -type parametermodify - modify the handle specfied with -handle parameter.enable - a list of handles can be used. Objects of type variable_descriptors cannot be enabled/disabled.disable - a list of handles can be used. Objects of type variable_descriptors cannot be enabled/disabled.remove - a list of handles can be used

DEFAULT

 None

-port_handle

Supported platforms Details

IxNetwork [M] , IxOS/IxNetwork-FT [M]

DESCRIPTION

Port to configure EFM organization_specific_info_tlv, variable_response_database or variable_descriptors on. (DEFAULT = N/A)

DEFAULT

 None

-os_info_tlv_oui

Supported platforms Details

IxNetwork, IxOS/IxNetwork-FT

DESCRIPTION

Three octets field in hex. Organizationally Unique Identifier for Organization Specific Info TLV. Use 0x format. Parameter -type must be 'organization_specific_info_tlv'.

DEFAULT

0x000000

-os_info_tlv_oui_step

Supported platforms Details

IxNetwork, IxOS/IxNetwork-FT

DESCRIPTION

Three octets field in hex. Step to increment Organizationally Unique Identifier when -count > 1.Use 0x format. Parameter -type must be 'organization_specific_info_tlv'. Incrementing will be done round robin.

DEFAULT

0x000001

-os_info_tlv_value

Supported platforms Details

IxNetwork, IxOS/IxNetwork-FT

DESCRIPTION

This field indicates the value of the Organization Specific Info TLV. This field's length is unspecified. Any hex value can be given (use 0x format). Parameter -type must be 'organization_specific_info_tlv'.

DEFAULT

0x00

-os_info_tlv_value_step

Supported platforms Details

IxNetwork, IxOS/IxNetwork-FT

DESCRIPTION

Increment step for -os_info_tlv_value when -count > 1. Any hex value can be given (use 0x format). Parameter -type must be 'organization_specific_info_tlv'.

DEFAULT

0x00

-reset

Supported platforms Details

IxNetwork

DESCRIPTION

If present deletes all objects of type -type created on port -port_handle before creating new ones. This parameter is supported using the following APIs: IxN.

DEFAULT

 None

-type

Supported platforms Details

IxNetwork, IxOS/IxNetwork-FT

DESCRIPTION

Not defined

Valid options are:

organization_specific_info_tlv

Add an organization specific info tlv

variable_response_database

Add a variable response database entry

variable_descriptors

Add a variable descriptor on the OAM

DEFAULT

organization_specific_info_tlv

-variable_branch

Supported platforms Details

IxNetwork, IxOS/IxNetwork-FT

DESCRIPTION

Variable Branch value in hex (1 octet, use 0x format). Parameter -type must be 'variable_response_database' or 'variable_descriptors'.

DEFAULT

0x00

-variable_branch_step

Supported platforms Details

IxNetwork, IxOS/IxNetwork-FT

DESCRIPTION

Step in hex to increment variable_branch value when -count > 1 (1 octet, use 0x format). Parameter -type must be 'variable_response_database' or 'variable_descriptors'.Incrementing will be done round robin. Only for -mode 'create'.

DEFAULT

0x01

-variable_indication

Supported platforms Details

IxNetwork, IxOS/IxNetwork-FT

DESCRIPTION

Indicate that this Leaf has some error. Valid options are:0 - false (DEFAULT)1 - trueParameter -type must be 'variable_response_database'. If this parameter is '1' parameters variable_width and variable_width_step are used and variable_value and variable_value_step are ignored.If this parameter is '0' parameters variable_width and variable_width_step are ignored and variable_value and variable_value_step are used.

DEFAULT

0

-variable_leaf

Supported platforms Details

IxNetwork, IxOS/IxNetwork-FT

DESCRIPTION

Variable Leaf value in hex (2 octets, use 0x format). Parameter -type must be 'variable_response_database' or 'variable_descriptors'.

DEFAULT

0x00

-variable_leaf_step

Supported platforms Details

IxNetwork, IxOS/IxNetwork-FT

DESCRIPTION

Step in hex to increment variable_leaf when -count > 1 (2 octets, use 0x format).Incrementing will be done round robin. Parameter -type must be 'variable_response_database' or 'variable_descriptors'. Only for -mode = 'create'.

DEFAULT

0x01

-variable_value

Supported platforms Details

IxNetwork, IxOS/IxNetwork-FT

DESCRIPTION

The variable_value parameter may be 1 to 128 octets in length. Its width is determined by the variable_width parameter. Hex value; use 0x format.Parameter -type must be 'variable_response_database'. This parameter is used only if -variable_indication is 0.

DEFAULT

0x00

-variable_value_step

Supported platforms Details

IxNetwork, IxOS/IxNetwork-FT

DESCRIPTION

Step to increment -variable_value when -count > 1. It may be 1 to 128 octets in length and must not exceed the length of variable_value.Incrementing will be done round robin. Parameter -type must be 'variable_response_database'. Only for -mode = 'create'. This parameter is used only if -variable_indication is 0.

DEFAULT

0x00

-variable_width

Supported platforms Details

IxNetwork, IxOS/IxNetwork-FT

DESCRIPTION

Configure the width for -variable_value. Parameter -type must be 'variable_response_database'. This parameter is used only if -variable_indication is 1.

DEFAULT

1

RETURN VALUES

Supported Platform Details

IxOS/IxNetwork-FT and IxNetwork

status

$::SUCCESS | $::FAILURE

log

On status of failure, gives detailed information.

handle

handle or handle list (if -count > 1)

EXAMPLES

IxOS/IxNetwork-FT and IxNetwork

See sample file efm_mixed_traffic_capture.tcl in Appendix A.

SAMPLE INPUT

SAMPLE OUTPUT
