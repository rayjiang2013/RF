ARGUMENTS

[M] indicates if the argument is mandatory on the supported platform.

-action_control

Supported platforms Details

IxNetwork [M]

DESCRIPTION

This is a mandatory argument. Used to start or stop for the action provided using parameter -action. This parameter is supported using the following APIs: IxTclNetwork.

Valid options are:

abort

Aborts all ANCP sessions including the underlying protocol.

bring_up_dsl_subscribers

Bring up DSL subscribers.

decoupled_start

Starts ANCP without starting any underlying protocol (deprecated).

decoupled_stop

Stops ANCP without tempering with the underlying protocol (deprecated).

start

Start ANCP emulation and underlying protocol (IP, PP, DHCP, etc).

start_adjacency

Start ANCP adjacency.

start_resync

Send port up message.

stop

Stop ANCP emulation and underlying protocol (IP, PP, DHCP, etc).

stop_adjacency

Stop ANCP adjacency.

tear_down_dsl_subscribers

Teardown DSL subscribers.

DEFAULT

 None

-action

Supported platforms Details

IxNetwork [M]

DESCRIPTION

This is a mandatory argument. Used to select the task to perform. Flapping is applicable for line subscriber devices only. The parameter details are as follows:

Valid options are:

send-reset

Not Supported.

enable

Enable ANCP protocol.

disable

Disable ANCP configuration.

reset

Not Supported.

flap_start

Not Supported.

flap_stop

Not Supported.

DEFAULT

 None

IxNetwork-NGPF [M]

DESCRIPTION

This is a mandatory argument. Used to select the task to perform. Flapping is applicable for line subscriber devices only. This parameter is supported using the following APIs: IxTclNetwork. Valid choices are: send-reset - Not Supported. enable - Enable ANCP protocol. disable - Disable ANCP configuration. reset - Not Supported. flap_start - Not Supported. flap_stop - Not Supported.

Valid options are:

enable

Supported platforms Details

DEFAULT

async

-ancp_handle

Supported platforms Details

IxNetwork

DESCRIPTION

DEFAULT

DEFAULT

 None

-batch_size

Supported platforms Details

IxNetwork

DESCRIPTION

Used to specify the number of instances executed per interval for ANCP Line subscriber only.

DEFAULT

 Not supported

-interval

Supported platforms Details

IxNetwork

DESCRIPTION

Used to specify the interval. By default it is in second. To change the unit use interval_unit option. Everything less than 1 second will be rounded to second. This is used for -action_control start_resync. This parameter is supported using the following APIs: IxTclNetwork.

DEFAULT

20

-interval_unit

Supported platforms Details

IxNetwork

DESCRIPTION

Used to specify the interval unit. This is used for -action_control start_resync. Choices: second | millisecond | microsecond.Everything less than 1 second will be rounded to second. This parameter is supported using the following APIs: IxTclNetwork.

DEFAULT

second

-iteration_count

Supported platforms Details

IxNetwork

DESCRIPTION

Used to specify the number of iterations. This is used for -action_control start_resync. This parameter is supported using the following APIs: IxTclNetwork.

DEFAULT

10

Supported platforms Details

IxNetwork

DESCRIPTION

Not supported

 Not supported
