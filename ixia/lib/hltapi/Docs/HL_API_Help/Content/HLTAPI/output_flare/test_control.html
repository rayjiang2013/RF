NAME - ::<namespace>:: test_control

DESCRIPTION

IxNetwork, IxOS/IxNetwork-FT

This command gives more options to control the various operations for a test session. Using this command all protocols configured in test sessions can be stop or start . This command gives more actions to control the QuickTest and also supports check_link_state. Valid only when using IxTclNetwork

IxNetwork-NGPF

This command gives more options to control the various operations for a test session. The command can be used to start or stop any (or all) protocols configured and to apply pending on the fly changes for NGPF protocols. The command gives control over quick tests and also supports check_link_state. Valid only when using IxTclNetwork (new API). Also this command is avaliable only in Ixia.

ARGUMENTS

[M] indicates if the argument is mandatory on the supported platform.

-action

Supported platforms Details

IxNetwork-NGPF [M]

DESCRIPTION

This option specified the action to be taken based on the option provided.

Valid options are:

start_all_protocols

Starts all the protcols configured on the ports.

stop_all_protocols

Stops all the protcols configured on the ports.

restart_down

Restarts all the sessions that are down on the given handles.

start_protocol

Starts protocols on given handles.

stop_protocol

Stops protocols on given handles.

abort_protocol

Aborts protocols on given handles.

apply_on_the_fly_changes

Applies all pending on the fly changes

check_link_state

Checks link state of all the ports used in the test.

get_all_qt_handles

Returns handles for all the quick tests existing in the current configuration.

get_available_qt_types

Return the list of all supported quick tests types.

get_qt_handles_for_type

Returns all the quick tests existing in the current configuraton of type specified in the parameter qt_type.

qt_remove_test

Removes the test specifed from the list of available configurations.

qt_apply_config

Apply the test configuration for the test specified in the parameter qt_handle.

qt_start

Start the test synchronously / asynchronously, depending on action_mode.

qt_run

Start the test synchronously/ asynchronously ,depending on action_mode, and wait for test finish or failure.

qt_stop

Stop the test synchronously / asynchronously, depending on action_mode

qt_wait_for_test

Waits for tests specified in the parameter qt_handle to finish or failure

is_done

Provide the status of asynchronous operations whether it is completed or not.

wait

Block/wait for the asynchronous operations to complete.

get_result

Provide detailed results of an asynchronous operation.

qt_get_input_params

Returns the list of all -inputParameters for the tests mentioned in qt_handle.

DEFAULT

 None

DESCRIPTION

check whether the ports are in up status.

check whether the ports are in down status.

check whether the ports are in unassigned status.

 None

Supported platforms Details

DESCRIPTION

Valid options are: sync - perform the operation specified in -action in synchronous mode async - perform the operation specified in -action in asynchronous mode (DEFAULT - sync) Valid when -action is qt_start, qt_stop, qt_run and qt_apply_config

Valid options are:

sync

async

Supported platforms Details

IxNetwork-NGPF

DESCRIPTION

(DEFAULT - continue) Valid when -action_mode sync and -action is qt_start, qt_stop, qt_run or qt_apply_config

Valid options are:

In case of failure in the current operation, stop the operation for remaining handles.

continue
