NAME - ::<namespace>:: cleanup_session

DESCRIPTION

IxNetwork, IxOS/IxNetwork-FT

This command disconnects from chassis, IxNetwork Tcl Server and Tcl Server, resets to factory defaults, and removes ownership from a list of ports. This command can be used after a script is run.

IxNetwork-NGPF

This command disconnects from chassis, IxNetwork Tcl Server and Tcl Server, resets to factory defaults, and removes ownership from a list of ports. This command can be used after a script is run.

-port_handle

Supported platforms Details

IxOS/IxNetwork-FT

DESCRIPTION

List ports to release.When using IxTclHal, IxTclProtocol or IxTclAccess, -port_handle option should always be present, otherwise only the disconnect part will be completed.

DEFAULT

 None

IxNetwork-NGPF

DESCRIPTION

List ports to release. When using IxTclHal, IxTclProtocol or IxTclAccess, -port_handle option should always be present, otherwise only the disconnect part will be completed.

DEFAULT

 None

-maintain_lock

Supported platforms Details

IxNetwork

DESCRIPTION

0 - Unlock test ports. Use this value at end of tests.1 - Do not unlock test ports. Use this value at beginning of tests.When using IxNetwork with the -reset option, this parameter will beignored.

DEFAULT

 None

IxNetwork-NGPF

DESCRIPTION

When using IxNetwork with the -reset option, this parameter will be ignored.

Valid options are:

1

Do not unlock test ports. Use this value at beginning of tests.

0

Unlock test ports. Use this value at end of tests.

DEFAULT

 None

DEPENDENCIES

Valid in combination with parameter(s)
'reset' | value= '0' |

-clear_csv

Supported platforms Details

IxNetwork, IxOS/IxNetwork-FT

DESCRIPTION

Not defined

Valid options are:

0

The CSV files are not deleted after cleanup_session procedure is called

1

The CSV files created after calling traffic_stats are deleted

DEFAULT

 None

IxNetwork-NGPF

DESCRIPTION

Valid choices are: 0 - The CSV files are not deleted after cleanup_session procedure is called 1 - The CSV files created after calling traffic_stats are deleted

DEFAULT

 None

-skip_wait_pending_operations

Supported platforms Details

IxNetwork, IxOS/IxNetwork-FT

DESCRIPTION

If there are any disconnect operations issued, this flag will prevent procedure from waiting for them to end. If this flag is used then there is no warranty that the disconnect operations have ended and exiting the script with these operations still running can be the source of errors.

DEFAULT

 None

IxNetwork-NGPF

DESCRIPTION

If there are any disconnect operations issued, this flag will prevent procedure from waiting for them to end. If this flag is used then there is no warranty that the disconnect operations have ended and exiting the script with these operations still running can be the source of errors.

DEFAULT

 None

-reset

Supported platforms Details

IxOS/IxNetwork-FT

DESCRIPTION

Reset the ports to factory defaults before releasing them.When using IxTclHal and IxTclProtocol, -port_handle option should be present also, otherwise the reset will not be completed.

DEFAULT

 None

IxNetwork-NGPF

DESCRIPTION

Reset the ports to factory defaults before releasing them. When using IxTclHal and IxTclProtocol, -port_handle option should be present also, otherwise the reset will not be completed.

DEFAULT

 None
