-mode

Supported platforms Details

IxNetwork, IxOS/IxNetwork-FT

DESCRIPTION

Supported with IxTclHal and IxTclNetwork (addAtmFilter choice is not valid with IxTclNetwork). When -mode is create all the filters, triggers and counters (general and ATM specific) will be configured.When -mode is addAtmTrigger, only the ATM specific configurations will be taken in consideration. This is useful when different configurations are needed for some handles

-capture_filter_DA

Supported platforms Details

IxNetwork, IxOS/IxNetwork-FT

DESCRIPTION

Supported with IxTclHal and IxTclNetwork. One of two available destination MAC addresses to filter on. Applicable only when capture_filter is set to 1.

Valid options are:

any

(default) disables the destination address filter constraint

DA1

sets the destination address filter constraint to trigger on frames with a destination MAC address that matches DA1and DA1mask as specified with ::<namespace>::packet_config_filter

notDA1

sets the destination address filter constraint to trigger on all frames except those with a destination MAC address that matches DA1 and DA1 mask as specified in the filter palette

DA2

sets the destination address filter constraint to trigger on frames with a destination MAC address that matches DA2 and DA2 mask

notDA2

sets the destination address filter constraint to trigger on all frames except those with a destination MAC address that matches DA2 and DA2 mask

DEFAULT

 None

-capture_filter_error

Supported platforms Details

IxNetwork, IxOS/IxNetwork-FT

DESCRIPTION

Supported with IxTclHal and IxTclNetwork (not all choices supported for IxTclNetwork). Applicable only when capture_filter is set to 1. Async triggers and uds1/uds2 can not be set when error type is other than errAnyFrame, errGoodFrame, errBadCRC, errBadFrame

Valid options are:

errAnyFrame

(default) disables the error filter constraint

errGoodFrame

sets the error filter constraint to trigger when frames with no errors are received

errBadCRC

sets the error filter constraint to trigger when frames with bad CRC errors are received

errBadFrame

sets the error filter constraint to trigger when corrupted frames are received

errAlign

not supported with IxTclNetwork. Sets the error filter constraint to trigger when frames with alignment errors are received (10/100 only)

errDribble

not supported with IxTclNetwork. Sets the error filter constraint to trigger when frames with dribble errors are received (10/100 only)

errBadCRCAlignDribble

not supported with IxTclNetwork. Sets the error filter constraint to trigger when frames with bad CRC, alignment error or dribble errors are received (10/100 only)

errLineError

not supported with IxTclNetwork. Sets the error filter constraint to trigger when frames with line errors are received (gigabit only)

errLineAndBadCRC

not supported with IxTclNetwork. Sets the error filter constraint to trigger line errors and bad CRC are received (gigabit only)

errLineAndGoodCRC

not supported with IxTclNetwork. Sets the error filter constraint to trigger when frames with line errors and bad CRC are received (gigabit only)

errUndersize

not supported with IxTclNetwork. Sets the error filter constraint to trigger when undersized frames (less than 64 bytes) are received

errOversize

not supported with IxTclNetwork. Sets the error filter constraint to trigger when oversized frames (greater than 1518 bytes) are received

errFragment

not supported with IxTclNetwork. Sets the error filter constraint to trigger when fragmented frames are received

errAnySequenceError

sets the error filter constraint to trigger when any of the next three conditions are true

errSmallSequenceError

sets the error filter constraint to trigger when the current sequence number minus the previous sequence number is less than or equal to the error threshold and not negative, or when the current sequence number is equal to the previous sequence number

errBigSequenceError

sets the error filter constraint to trigger when the current sequence number minus the previous sequence number is greater than the error threshold

errReverseSequenceError

sets the error filter constraint to trigger when the current sequence number is less than the previous sequence number

errDataIntegrityError

sets the error filter constraint to trigger when any data integrity error is detected

errGfpErrors

not supported with IxTclNetwork. Sets the error filter constraint to trigger when any GFP error is detected. The particular errors that are used are controlled by options of the ::<namespace>::packet_config_filter.

errCdlErrors

not supported with IxTclNetwork. Sets the error filter constraint to trigger when any CDL preamble error is detected

errAnyIpTcpUdpChecksumError

not supported with IxTclHal

errInvalidFcoeFrame

not supported with IxTclHal

DEFAULT

 None
