NAME - ::<namespace>:: emulation_stp_vlan_config

DESCRIPTION

IxNetwork, IxOS/IxNetwork-FT

This command is used to create/modify/delete/enable/disable a STP VLAN object. This object can be created on a PVST or RPVST bridge object. After this object is created it should be assigned to bridge - with mode modify of ixia

ARGUMENTS

[M] indicates if the argument is mandatory on the supported platform.

-bridge_handle

Supported platforms Details

IxNetwork

DESCRIPTION

Bridge handle on which to create VLAN object. This parameter must be specified in create mode. This parameter should be specified in create mode.

DEFAULT

 None

-count

Supported platforms Details

IxNetwork

DESCRIPTION

Number of VLAN objects to create for bridge specified by handle. This parameter is valid when -mode is create.

DEFAULT

1

-handle

Supported platforms Details

IxNetwork

DESCRIPTION

VLAN component of bridge handle to be modified. One of the two parameters is required: port_handle/handle. This option is valid for every mode except create.

DEFAULT

 None

-internal_root_path_cost

Supported platforms Details

IxNetwork

DESCRIPTION

The administrative cost for the shortest path from this bridge to the Root bridge. A 4-byte unsigned integer. This option is valid only when: -mode is create or -mode is modify and -handle is provided.

DEFAULT

0

-mode

Supported platforms Details

IxNetwork

DESCRIPTION

The action to take on the port_handle/handle provided. For mode create port_handle shoud be specified. For others handle should be provided.

DEFAULT

create

-root_mac_address

Supported platforms Details

IxNetwork

DESCRIPTION

The 6-byte MAC address for the Root bridge. This option is valid only when -mode create and -bridge_handle is provided or -mode modify and -handle is provided.

DEFAULT

0000.0000.0000

-root_mac_address_step

Supported platforms Details

IxNetwork

DESCRIPTION

Specify the step for root_mac_address between STP VLAN objects. This option is valid only when -mode create and -bridge_handle is provided.

DEFAULT

0000.0000.0001

-root_priority

Supported platforms Details

IxNetwork

DESCRIPTION

The priority of the port. Since MAC address reduction is used, only multiples of 4096 are used. Choose one of the following values from the list:0 4096 8192 12288 16384 20480 24576 28672 32768 36864 40960 45056 49152 53248 57344 61440 This option is valid only when -mode create and -bridge_handle is provided or -mode modify and -handle is provided.

DEFAULT

 None

-vlan_port_priority

Supported platforms Details

IxNetwork

DESCRIPTION

The Port Priority for the emulated port on this PVST+ or RPVST+ bridge that is connected to the VLAN. A 2-byte unsigned integer. The valid range is 0 to 63. This option is valid only when -mode create and -bridge_handle is provided or -mode modify and -handle is provided.

DEFAULT

32

-vlan_port_priority_step

Supported platforms Details

IxNetwork

DESCRIPTION

The step for the Port Priority for the emulated port on this PVST+ or RPVST+ bridge that is connected to the VLAN. A 2-byte unsigned integer. The valid range is 0 to 63. This option is valid only whenThis option is valid only when: -mode is create or -mode is modify and -handle is provided.

DEFAULT

1

-vlan_id

Supported platforms Details

IxNetwork

DESCRIPTION

This option is valid only when -mode create and -bridge_handle is provided or -mode modify and -handle is provided. (DEFAULT = 2 for first item, 3 for second, etc.)

DEFAULT

2

RETURN VALUES

Supported Platform Details

IxOS/IxNetwork-FT and IxNetwork

status

$::SUCCESS or $::FAILURE

log

When status is failure, contains more information

handle

The list of bridges created
