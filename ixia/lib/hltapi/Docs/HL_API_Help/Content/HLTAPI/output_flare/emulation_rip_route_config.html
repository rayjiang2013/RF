Supported platforms Details

-num_prefixes

Supported platforms Details

-prefix_start

Supported platforms Details

IxNetwork, IxOS/IxNetwork-FT

DESCRIPTION

First route (prefix) to be advertised.

DEFAULT

 None

-prefix_length

Supported platforms Details

IxNetwork, IxOS/IxNetwork-FT

DESCRIPTION

Network mask width. RANGE 1-128.

DEFAULT

 None

-prefix_step

Supported platforms Details

IxNetwork, IxOS/IxNetwork-FT

DESCRIPTION

How to increment prefixes. Default is using prefix length (minimum step). The value would be rounded to the prefix length.

DEFAULT

 None

-metric

Supported platforms Details

IxNetwork, IxOS/IxNetwork-FT

DESCRIPTION

The total cost for these routes. RANGE 0-4,294,967,295.

DEFAULT

 None

-next_hop

Supported platforms Details

IxNetwork, IxOS/IxNetwork-FT

DESCRIPTION

The immediate next hop IP address on the way to the destination prefix. Used by RIPv2 & RIPng, default is all zeros IP address which indicates the next hop is update source address.

DEFAULT

 None

-route_tag

Supported platforms Details

IxNetwork, IxOS/IxNetwork-FT

DESCRIPTION

An arbitrary value associated with the routes in this range. Used by RIPv2 & RIPng. RANGE 0-65535.

DEFAULT

 None

Supported Platform Details

IxOS/IxNetwork-FT and IxNetwork

status

$::SUCCESS | $::FAILURE

log

When status is failure, contains more information

route_handle

The list of route range numbers added
