NAME - ::<namespace>:: emulation_ftp_traffic_config

DESCRIPTION

IxLoad

This command is used to construct the model for client or server traffic to be applied, or respectively, handled during the test. This traffic is composed by commands created by client FTP agents which are handled by server agents.Multiple agents can be added to form a FTP traffic.

ARGUMENTS

[M] indicates if the argument is mandatory on the supported platform.

-handle

Supported platforms Details

IxLoad

DESCRIPTION

Handle of a FTP configuration when -property traffic and -mode add. Or handle of a traffic property , used when -property is agent and -mode is add. Also the handle of a traffic or agent property when -property is respectively traffic, agent and -mode is modify | remove | enable | disable.

DEFAULT

 None

-target

Supported platforms Details

IxLoad

DESCRIPTION

Specifies where this traffic will be added: on the client side or on the server side. Values: client or server. Dependencies: -property traffic with -mode add.

DEFAULT

 None

-property

Supported platforms Details

IxLoad

DESCRIPTION

Defines types of properties of the FTP traffic configuration.

Valid options are:

traffic

handles operations required for defining the FTP

agent

handles operations required for defining the emulated

DEFAULT

 None

-mode

Supported platforms Details

IxLoad [M]

DESCRIPTION

Defines types of actions to be taken on the -property object. Values: add | remove | modify | enable | disable.

DEFAULT

 None

-esm_enable

Supported platforms Details

IxLoad

DESCRIPTION

If true, the use of the esm option is enabled. Dependencies: property agent with mode add | modify

DEFAULT

 None

-esm

Supported platforms Details

IxLoad

DESCRIPTION

If esm_enable is true, this option specifies the TCP Maximum Segment Size in the MSS (RX) field. Otherwise, the TCP Maximum Segment Size is 1460 bytes.Dependencies: esm_enable 1 ; property agent with mode add | modify

DEFAULT

 None

-ftp_mode

Supported platforms Details

IxLoad

DESCRIPTION

The mode that the client will use to access the server: To establish an FTP connection, the client connects from a random unprivileged port (port n, where n is greater than 1024) to the FTP server's command port, normally port 21. What happens next depends on whether the client is in active or passive mode.Dependencies: target client ; property agent with mode add | modifyactive - (default) The client sends the PORT command and waits for an OK response from the server.passive - In Passive mode, the client initiates both connections to the server.

DEFAULT

 None

-ftp_port

Supported platforms Details

IxLoad

DESCRIPTION

The port number that the server will respond on.Dependencies: target server ; property agent with mode add | modify

DEFAULT

 None

-arguments

Supported platforms Details

IxLoad

DESCRIPTION

This option contains an argument which is used by the various commands defined in the command option. The type of the value depends on the command:cd = the path to switch toget = the path to where the file is stored on the serverput = the path/size of the file to be sent to the serverretrieve = the path to where the file is storedstore = the path/size of the file to be sent to the serverthink = the number of milliseconds to pause before executing the next actionDependencies: property action with mode add | modify

DEFAULT

 None

-command

Supported platforms Details

IxLoad

DESCRIPTION

Selects the FTP command to be used:cd = Changes the current working directory to the value in the arguments option.get (DEFAULT) = Retrieves the file specified in the arguments option. {Get} is not a standard FTP command. It allows you to retrieve a file from a server without having to log in.login = Logs in to the FTP server using the name and password in the userName and password optionsput = Copies the file specified in the arguments option from the client to the server. {Put} is not a standard FTP command. It allows you to store a file on a server without having to log inquit = Logs out of the FTP server.retrieve = Retrieves the file specified in the arguments option.store = Copies the file specified in the arguments option from the client to the server.think = Adds a pause (think time) before the next command is executed. Specify the duration of the pause in the arguments option.loop_begin = An IxLoad command that you can add to the Command List to cause the commands between it and the {Loop End} to be executed a specified number of times.loop_end = Ends the list of commands that will be executed by the preceding {Loop Begin} commandDependencies: property action with mode add | modify

DEFAULT

 None

-destination

Supported platforms Details

IxLoad

DESCRIPTION

The IP address of a real FTP server. If the FTP server listens on a port other than the standard (21), enter a colon after the IP address and then enter the port number. When using an FTP Server Agent this option will not be used, you must provide a FTP server agent handler to the agent_handler option instead. The port number must agree with that defined by the Server Agent. For example: 192.168.0.1:21Dependencies: ssl_enable 1 ; Dependencies: property action with mode add | modify

DEFAULT

 None

-user_name

Supported platforms Details

IxLoad

DESCRIPTION

The user name that the client will use to log in to the FTP server. The emulated servers only accept a user name of root.Dependencies: property action with mode add | modify

DEFAULT

 None

-password

Supported platforms Details

IxLoad

DESCRIPTION

The password for the user name.Dependencies: Dependencies: property action with mode add | modify

DEFAULT

 None

-agent_handler

Supported platforms Details

IxLoad

DESCRIPTION

The handler of a FTP server agent if destination is a emulated FTP server.Dependencies: property action with mode add | modify

DEFAULT

 None

RETURN VALUES

Supported Platform Details

IxLoad

status

$::SUCCESS | $::FAILURE

log

When status is $::FAILURE, contains more information

handles

the handles of a FTP traffic configuration, list of traffic, agent.

EXAMPLES

IxLoad

See files starting with ftp_ in the Samples subdirectory. Also see the FTP example in Appendix A, "Examples."
