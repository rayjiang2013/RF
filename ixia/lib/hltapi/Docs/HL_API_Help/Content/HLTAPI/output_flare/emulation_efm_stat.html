ARGUMENTS

-port_handle

Supported platforms Details

IxNetwork [M] , IxOS/IxNetwork-FT [M]

DESCRIPTION

Port to get EFM stats from. (DEFAULT = N/A)

DEFAULT

 None

-action

Supported platforms Details

IxNetwork [M] , IxOS/IxNetwork-FT [M]

DESCRIPTION

Specify what to do with the EFM stats for the port, either get them or reset.When IxOS TCL API implementation is used it is mandatory to start port capture in order to retrieve EFM statistic keys marked with alarm, otherwise these statistics are N/A (for IxNetwork/IxProtocol implementation this is not required). When IxOS TCL API implementation is used, inspecting capture buffer to provide alarm statistics can be suppressed using the skip_capture_stats flag.

Valid options are:

get

Get statistics.

reset

Reset statistics. If ::<namespace>::traffic_config -action clear_stats

DEFAULT

 None

-api_used

Supported platforms Details

IxNetwork, IxOS/IxNetwork-FT

DESCRIPTION

Not defined

Valid options are:

ixnetwork

Use IxNetwork implementation. Valid only if the HLTSET used loads IxNetwork TCL API.

x

Use IxNetwork implementation. Valid only if the HLTSET used loads IxNetwork TCL API.

ixprotocol

Use IxProtocol implementation. Valid only if the HLTSET used loads IxProtocol TCL API.

ixos

Use IxOS implementation. This is the default if the HLTSET does not load

DEFAULT

 None

-capture_file

Supported platforms Details

IxNetwork

DESCRIPTION

Path to Ixia compatible capture file. If this parameter is present, alarm statistics will be taken from the capture file. Otherwise the capture buffer from the -port_handle is inspected. If the port capture buffer is empty and no capture_file was specified the alarm statistic keys will have N/A values. Not available with IxNetwork/IxProtocol Tcl API.

DEFAULT

 None
