IxAccess to IxNetwork Migration

This appendix describes the differences between IxNetwork and IxAccess regarding PPPoX and L2TPoX + traffic. This information is useful when converting scripts. Side-by-side examples of PPPoX and L2TPoX scripts can be examined in Sample Scripts IxAccess to IxNetwork Migration..

  • :IxNetwork refers to HLTAPI configurations done using the IxNetwork TCL API.
  • IxAccess refers to HLTAPI configurations done using IxAccess Tcl API.

Summary of Differences PPPoX and Traffic. provides an overall summary of the differences for PPPoX and Traffic. Summary of Differences L2TPoX. provides a summary of the differences for L2TPoX.

For details, refer to the following subsections:

  • PPPoX and Traffic Differences.

    • ::ixia::pppox_stats.

    • l2tp_config.
    • l2tp_control.
    • l2tp_stats.

Key
  • Not Supported: the number of parameters that are no longer supported.
  • Partially Supported: the number of parameters that are still supported but some of the functionality will not be available (for example, the range of accepted values is modified or some of the choices are not supported).
  • Different Functionality: the number of parameters that still exist but do not have the same functionality as they did with IxAccess (for example, -handle parameter in pppox_stats will not grab pppox statistics for the pppox_handle--it will provide statistics for the port on which the pppox_handle was configured).
  • New: new parameters that bring new functionality without breaking the old functionality.
  • Statistic Keys N/A: statistic keys that are no longer supported.
  • Statistic Keys NEW: statistic keys that are new.
