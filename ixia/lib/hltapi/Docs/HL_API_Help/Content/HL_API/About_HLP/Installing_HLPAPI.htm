Installing HLPAPI

Windows

The HLPAPI package is available in the form of an executable file:

HLPAPI_<release>.exe

where <release> is the release designation.

To install HLPAPI, do the following:

  1. From the Ixia Website, download the .exe file to your computer.
  2. Double-click this file to start the setup menu for HLPAPI installation. Follow the setup menu to complete the installation.

To access the library procedures, do the following:

  • Use the following command:

package require Ixia

UNIX

To install HLPAPI on a Linux and Solaris computer, do the following:

  1. Download either the ZIP or TAR file from the portal.
  2. Extract and copy all folders and files from the ZIP or TAR file into the lib folder of the IxOS client installation. For example, if the IxOS client is installed in /home/ixia/ixos6.40, the HLPAPI should be extracted to /home/ixia/ixos6.40/lib.
    Alternatively, create a HLPAPI only folder and extract the files there. In this case, set the TCLLIBPATH to point to both IXIA client installation and the HLPAPI installation.
  3. Add the following environment variables to point to the HLPAPI package:
  • IXIA_HOME: where the IxOS client is installed.
  • IXIA_VERSION–HLTSET value: This is one of several preset options. For more information on how to set this variable, see Setting the IXIA_VERSION Variable.
  • TCLLIBPATH: path to the IxTclHal and HLPAPI library where the package can be found.

See the following example:

IXIA_HOME=/home/ixia/ixos4.0

IXIA_VERSION=HLTSET8

TCLLIBPATH=$IXIA_HOME/lib

If HLPAPI is installed in its own folder, set TCLLIBPATH to also point to the HLPAPI folder. For example:

HLPAPI_HOME=/home/HLPAPI2.4

TCLLIBPATH="$IXIA_HOME/lib $HLPAPI_HOME"

To access HLPAPI library procedures, issue the following commands in Tcl or expect shell:

package require Ixia

  • LD_LIBRARY_PATH: folders to search for shared libraries.

LD_LIBRARY_PATH=$IXIA_HOME/lib:$TCL_HOME/lib:$LD_LIBRARY_PATH

PATH=$IXIA_HOME/bin:$TCL_HOME/bin:$PATH

For HLT Perl API,

The Perl version installed by HLT can be accessed using the ixiaperl command, which is a symbolic link created while installing HLT. In case you have installed multiple versions of Perl, the ixiaperl command runs the highest version of Perl installed.

To run your script using the highest version of Perl installed use the following syntax:

ixiaperl <script name>

To run your script using a different version of Perl, use the ixiaperl command from the specific Perl installation folder. You can use the following syntax:

~/Ixia/<hltapi version 1>/perl/bin/ixiaperl <script name>

HLPAPI requires the following Tcl lib packages:

  • Tclx: this is used for keyed lists. The HLT commands need this to return a keyed list. This library is usually included in third-party Tclsh like Active Tcl 8.4.
  • Mpexpr: this is used for IPv6 addresses’ operations. IxOS client package includes this library with the Tcl 8.4 package. If you use third-party tclsh, you have to ensure that mpexpr is compiled with the third-party tclsh.
  • Snit: this is used for auto generated protocols (FCoE, ESMC, and DCBX).

.The default setting for env (IXIA_VERSION) is HLTSET158. If the value is set to an IxOS major.minor version, the largest HLTSET list that corresponds to that IxOS version is used.

A new HLTSET is introduced, only for IxNetwork. Previous HLTSETs are (P) and (N). The (P) version uses IxTclHal (IxOS) and IxTclProtocol (IxNetwork). The (N) version uses IxTclHal (IxOS) and IxTclNetwork (IxNetwork). The new HLTSET (NO) uses only IxTclNetwork and has improved performance in chassis connection and port assignment.

Meaning of HLTSETs

  • P - Loads:
    • IxOS (tclHal API)
    • IxTclProtocols
  • N - Loads:
    • IxOS (tclHal API)
    • IxTclNetwork
  • NO - Loads:
    • IxTclNetwork (only, without IxTclHal)
  • P2NO - Loads:
    • If -ixnetwork_tcl_server parameter is NOT provided in ixiahlt::connect (same as P)
    • IxOS (tclHal API)
    • IxTclProtocols
  • If -ixnetwork_tcl_server parameter is provided in ixiahlt::connect (same as NO)
    • IxTclNetwork (only; without IxTclHal)
