Command Conventions

All the commands in this guide are in the following format:

command_name_example

If the command is made up of multiple words, they are separated by an underscore (‘_’).

In a few instances in this guide, because of space constraints, commands are forced to break in the middle of the word. When this happens, ignore the dash indicating the word break when using the command. For example, if the following command were shown in the guide:

command_name_ex-ample

it should be used as follows:

command_name_example

If the command breaks at an underscore, it is still necessary to include the underscore in the command. For example, if the following command were shown in the guide:

command_name_example

It should be used as follows:

command_name_example

Some commands in this guide are in colored text. Red commands are Ixia specific commands, while blue commands are unsupported commands. For example:

ipv6_mpls_nlri

is an Ixia specific command, while

remote_as

is an unsupported command.

All the parameters within square brackets are considered optional. Mandatory parameters are not displayed inside square brackets.

For sample scripts of HLP API refer to the following path:
For Linux: <HLT API Install Folder>/Samples/Perl
For Windows: C:\Program Files\Ixia\hltapi\<HLT API Version>\TclScripts\lib\ hltapi\Samples\Perl
