Debugging HLPython Scripts

This section describes the debugging techniques for HLPython scripts and not restricting the user in using any other desired debugger.

As HLPython scripts behave similarly as other Python language scripts, therefore a direct debugging approach is employed by using Python’s interactive source code debugger defined by the pdb module.

This method involves the following functionalities:

  1. Importing the pdb module in the script.
  2. Setting breakpoints by inserting pdb.set_trace() at the location where the user wants to break into the debugger.
  3. Stepping through the code using the pdb specific commands.

Refer to the official website of Python documentation at www.python.org for further information on the usage of pdb.

As an assistant in developing and debugging HLPython scripts, Ixia provides a “help” functionality, acting as a manual page for the commands available under the IxiaNgpf module.

In order to access the help tool, the user is required to open the Python interpreter and instantiate the ixiangpf object, as shown in the Guidelines for Creating a HLPython Script section, and then call help(ixiangpf.<command_name>) in order to get information on a specific HL command.
