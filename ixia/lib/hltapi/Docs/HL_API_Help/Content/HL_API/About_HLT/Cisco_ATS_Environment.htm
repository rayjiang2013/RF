ATS Environment

ATS users can start using HLTAPI without installing the software themselves. Under ATS, Ixia client software is installed at one location of a Solaris file server. The software includes IxTclHal, IxTclProtocol, IxTclAccess, and others, as well as HLTAPI. The software is accessed through file mounting points, so that Ixia software appears as local files.

The environment variable IXIA_VERSION should be set to HLTSET values, which is a predefined set of values for each of the HLTAPI compatible products. The HLTSET values for the release can be found in the release notes, and are listed in a table in the Setting the IXIA_VERSION Values section. This enables HLTAPI to load the correct Ixia client software.

Although you do not need to install HLTAPI, it is useful to know where to find it. To get to the HLTAPI folders, change the folder (using the cd command) to $AUTOTEST/lib/ixia/hltapi. Several HLTAPI distributions are here, for example:

There are three important subfolders. They are as follows:

  • Docs/, where you can find the HLTAPI manual in PDF format.
  • Samples/, where there are many sample files covering IPv4, IPv6, and various interface types and protocols.
  • library/, where the source files for HLTAPI are stored. They are the ultimate source of information.

For convenience, it is possible to make a symbolic link to the HLTAPI folder by:

ln -s $AUTOTEST/ lib/ixia/hltapi/2.60 ~/hltapi

Access to the file is now possible by referring to ~/hltapi.
