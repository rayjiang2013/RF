Installing HLTAPI

Windows

The HLTAPI package is provided in the form of an executable file:

HLTAPI_ <release> .exe

where <release> is the release designation.

To install the HLTAPI, do the following:

  1. Download the .exe file from the portal to your computer.
  2. Double-click this file to start the setup menu for HLTAPI installation. Follow the setup menu to complete the installation.

To access the library procs, use the following command:

package require Ixia

UNIX

To install the HLTAPI on a Linux/Solaris computer, do the following:

  1. Download either the ZIP or TAR file from the portal.

  2. Extract and copy all folders and files from the ZIP or TAR file into the lib folder of the IxOS client installation. For example, if the IxOS client is installed in /home/ixia/ixos4.70, the HLTAPI should be extracted to /home/ixia/ixos4.70/lib.

    Alternatively, create a HLTAPI only folder and extract the files there. In this case, set the TCLLIBPATH to point to both IXIA client installation and the HLTAPI installation.

  3. Add the following environment variables to point to the HLTAPI package:
    • IXIA_HOME-where the IxOS client is installed.
    • IXIA_VERSION-HLTSET value. This is one of several preset options. Refer to Setting the IXIA_VERSION Variable. for more information on how to set this variable.
    • TCLLIBPATH-path to the IxTclHal and HLTAPI library where the package can be found.

See the following example:

IXIA_HOME=/home/ixia/ixos4.0

IXIA_VERSION=HLTSET8

TCLLIBPATH=$IXIA_HOME/lib

If HLTAPI is installed in its own folder, set TCLLIBPATH to also point to the HLTAPI folder. For example:

HLTAPI_HOME=/home/hltapi2.4

TCLLIBPATH="$IXIA_HOME/lib $HLTAPI_HOME"

To access HLTAPI library procs, issue the following commands in Tcl or expect shell:

package require Ixia

LD_LIBRARY_PATH-directories to search for shared libraries.

LD_LIBRARY_PATH=$IXIA_HOME/lib:$TCL_HOME/lib:$LD_LIBRARY_PATH

PATH=$IXIA_HOME/bin:$TCL_HOME/bin:$PATH

 

For HLT Perl API,

LD_LIBRARY_PATH=$IXIA_HOME/lib:$PERL_HOME/lib:$LD_LIBRARY_PATH

PATH=$IXIA_HOME/bin:$PERL_HOME/bin:$PATH

 

TCL_HOME is where the TCL is installed.

PERL_HOME is where the PERL is installed.

 

HLTAPI requires other Tcl lib packages. See below:

  • Tclx - this is used for keyed lists. The HLT commands need this to return a keyed list. This library is usually included in third party Tclsh like Active Tcl 8.4.
  • Mpexpr - this is used for IPv6 addresses operations. IxOS client package includes this library with Tcl 8.4 package. If you use third party tclsh, you have to make sure that mpexpr is compiled in with the third party tclsh.
  • Snit - this is used for auto generated protocols (FCoE, ESMC, DCBX and so on.)

Running HLTAPI from a client windows machine using remote installation repository

Currently, HLTAPI retrieves the AppInfo, the IxNetwork and IxOS paths and versions from the windows registry. The client wants to run HLTAPI scripts without installing IxNetwork and IxOS on the local machine. IxNetwork and IxOS will be installed on a remote machine, and HLTAPI has to access those paths in order to run the scripts.

When HLTAPI is installed on a Windows machine, it retrieves the AppInfo, the IxNetwork and IxOS paths and versions from the Windows registry. In order to run HLTAPI scripts without installing IxNetwork and IxOS on the local machine, IxNetwork and IxOS will be installed on a remote machine, and HLTAPI has to access the remote machine paths in order to run the scripts.

The topology diagram is as follows:

User can run the scripts from the 'NO IXIA SOFTWARE' machine. The Ixia software (IxNetwork, IxOS, HLTAPI, IxLoad) is installed on the File server.

The Active TCL console from the 'NO IXIA SOFTWARE' machine has to load the Ixia packages from the File server.

To run HLTAPI from a client windows machine using remote installation repository, you need to follow the steps as described below:

  1. Install “Microsoft Visual C++ 2010 Redistributable Package” on the “NO IXIA SOFTWARE” machine.

  2. Open Tcl console and manually append the File Server’s IxNetwork, IxOS, IxNProtocol, HLTAPI paths to env(PATH) and ::auto_path.

    The process is explained with an example as follows:

    • append env(PATH) {Z:\IxNProtocols\7.10.0.622;Z:\IxNetwork\7.10.0.100-EB;Z:\IxOS\6.40-EA;Z:\IxOS\6.40-EA/../..;Z:\Perl\site\bin;Z:\Perl\bin}
    • append ::auto_path " \
    • {Z:/IxOS/6.40-EA/TclScripts/lib/IxTcl1.0} \
    • {Z:/IxOS/6.40-EA/TclScripts/lib/ixTclServices} \
    • {Z:/hltapi/4.70.0.69/TclScripts/lib/hltapi/library/common/ixia_hl_lib-7.10} \
    • {Z:/IxNProtocols/7.10.0.622/TclScripts/lib} \
    • {Z:/IxNetwork/7.10.0.100-EB/TclScripts/lib} \
    • {Z:/IxOS/6.40-EA/../../TclScripts/lib} \
    • {Z:/IxOS/6.40-EA/TclScripts/lib} \
    • {Z:/IxOS/6.40-EA/TclScripts/lib/IxTcl1.0} \
    • {Z:/hltapi/4.70.0.69/TclScripts/lib}"

  3. 'package require Ixia'

All versions mentioned above are just an example and should be replaced with the correct user versions.

Determine what version of the product to be tested is used. Use the value in the far left column as the value for IXIA_VERSION. If multiple products are used, be sure to use a value that incorporates the appropriate version of all Ixia products in the test.

The default setting for env (I.XIA_VERSION) is HLTSET158. If the value is set to an IxOS major.minor version, then the largest HLTSET list that corresponds to that IxOS version is used.

A new HLTSET is introduced for IxNetwork only. Previous HLTSETs are (P) and (N). The (P) version is using IxTclHal (IxOS) and IxTclProtocol (IxNetwork). The (N) version is using IxTclHal (IxOS) and IxTclNetwork (IxNetwork). The new HLTSET (NO) uses IxTclNetwork only and has improved performance in chassis connection and port assignment.

Meaning of HLTSETs

  • P - Loads:
    • IxOS (tclHal API)
    • IxTclProtocols
  • N - Loads:
    • IxOS (tclHal API)
    • IxTclNetwork
  • NO - Loads:
    • IxTclNetwork (only; without IxTclHal)
  • P2NO - Loads:
    • If -ixnetwork_tcl_server parameter is NOT provided in ::ixia::connect (same as P)
      • IxOS (tclHal API)
      • IxTclProtocols
  • If -ixnetwork_tcl_server parameter is provided in ::ixia::connect (same as NO)
    • IxTclNetwork (only; without IxTclHal)

Known Limitations

The known limitations are:

  • In the traffic_config call, ICMP options are valid for IPv4 only; ICMP options are not currently supported for IPv6 streams.
  • IGMP is only supported on ports that contain a CPU.
