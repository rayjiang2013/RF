Perl HLT Bindings

Available Features for Importing ixiatcl, ixiahlt, and ixiaixn

The ixiahlt and ixiaixn modules can be supplied with an additional hashtable argument when importing the package through ’use.’

This hash specifies various features that the package should adhere to when importing the package.

ixiatcl

use ixiatcl {

IXIAPERLSERVER => '10.10.20.6:5678',

TclAutoPath => ['c:/tcl/libA', 'c:/tcl/libB']

};

  • IXIAPERLSERVER => hostname:portnumber

(Default value is the empty string, which means do not use a Perl server)

: You can also enable this through an environment variable of the same name. The environment variable will override any value declared in the use statement.

  • TclAutoPath => [array of folder names that contain tcl libraries to add to tcl's ::auto_path]

(Default is an empty array)

ixiaixn

use ixiaixn {

IxTclNetworkConnector => 0,

TclAutoPath => ['c:/tcl/libA', 'c:/tcl/libB']

};

  • IxTclNetworkConnector => 0 | 1

(Default value is 0) use this only once. Do not use it in both. Use ixiahlt AND use ixiaixn

  • TclAutoPath => [array of folder names that contain tcl libraries to add to tcl's ::auto_path]

(Default is an empty array)

ixiahlt

use ixiahlt {

IXIA_VERSION => 'HLTSET99',

IxTclNetworkConnector => 0,

TclAutoPath => ['c:/tcl/libA', 'c:/tcl/libB']

};

  • IxTclNetworkConnector => 0 | 1

(Default value is 0) use this only once. Don't use it in both use ixiahlt AND use ixiaixn

  • TclAutoPath => [array of folder names that contain tcl libraries to add to tcl's ::auto_path]

(Default is an empty array)
