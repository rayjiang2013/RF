[M] indicates if the argument is mandatory on the supported platform.

-handle

Supported platforms Details

IxNetwork-NGPF

DESCRIPTION

The handle of a protocol stack or another handle returned by a prior tlv_config command.

DEFAULT

 None

-tlv_handle

Supported platforms Details

IxNetwork-NGPF

DESCRIPTION

The handle of a TLV from a default or custom TLV Template that will be used to define the structure of a new TLV that is added to the TLV Profile of a protocol.

DEFAULT

 None

-protocol

Supported platforms Details

IxNetwork-NGPF

DESCRIPTION

The type of protocol that the TLV applies to.

Valid options are:

dhcp4_client

TLV configuration for DHCPv4 Clients

dhcp4_server

TLV configuration for DHCPv4 Servers

dhcp6_client

TLV configuration for DHCPv6 Clients

dhcp6_server

TLV configuration for DHCPv6 Servers

dhcp4_relay

TLV configuration for DHCPv4 Relay Agents

dhcp6_relay

TLV configuration for DHCPv6 Relay Agents

dhcp6_relay_light

TLV configuration for lightweight DHCPv6 Relay Agents

ancp

TLV configuration for Ancp

DEFAULT

 None

-mode

Supported platforms Details

IxNetwork-NGPF [M]

DESCRIPTION

The action that the command will execute.

Valid options are:

create_template_group

Create a new custom TLV Template group that will be used as a custom container for a series or TLVs.

create_tlv

Create a new TLV or sub TLV in a custom TLV Template or add a new TLV to the TLV Profile of a protocol.

create_tlv_container

Creates a new TLV container that can be used to add a group of nested sub TLVs to a custom TLV Template.

create_field

Creates a new field in a TLV from a TLV Template or the TLV Profile of a protocol.

modify

Modifies an existing custom TLV Template, container, field, TLV or sub TLV.

delete

Deletes an existing custom TLV Template, container, field, TLV or sub TLV.

delete_all

Deletes all existing TLVs from the TLV Profile of specified protocol.

DEFAULT

 None

-template_group_name

Supported platforms Details

IxNetwork-NGPF

DESCRIPTION

When the -mode is create_template_group the value of this argument will be used to specify the name of the new TLV Template group. When the -mode is create_tlv the value of this argument will be used to find an existing TLV Template group with a matching name.

DEFAULT

 None
