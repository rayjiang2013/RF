NAME - ::<namespace>:: emulation_bfd_session_config

DESCRIPTION

IxNetwork, IxOS/IxNetwork-FT

This command is used to create/modify/delete/enable/disable a BFD session on a BFD router interface previously created with ::::emulation_bfd_config. One BFD session can be added to all router interfaces or just to a specific router interface.

ARGUMENTS

[M] indicates if the argument is mandatory on the supported platform.

-count

Supported platforms Details

IxNetwork

DESCRIPTION

The number of BFD sessions to create for a provided -handle parameter. If the provided -handle is a list of BFD routers, then -count sessions will be created for each BFD router interface. If the provided -handle is a list of BFD router interface, then -count sessions will be created for each BFD router interface in the list.This option is valid only when -mode is create and -handle is provided.

DEFAULT

1

-enable_auto_choose_source

Supported platforms Details

IxNetwork

DESCRIPTION

Setting this parameter to 1, enables the session to automatically choose the source IP address for the BFD session. In this case, the local_ip_addr parameter is ignored. Setting this parameter to 0, enables the local_ip_addr parameter.This option is valid only when -mode is create and -handle is provided or -mode is modify and -session_handle is provided.

DEFAULT

1

-enable_learned_remote_disc

Supported platforms Details

IxNetwork

DESCRIPTION

Enables the ability of this BFD session to learn the value of the discriminator of the far end of the BFD session. If this option is set to 1, the remote_disc option will be ignored.This option is valid only when -mode is create and -handle is provided or -mode is modify and -session_handle is provided.

DEFAULT

1

-handle

Supported platforms Details

IxNetwork

DESCRIPTION

This is the router handle or the router interface handle that needs to be provided when creating a BFD session. This is mandatory when -mode is create.

DEFAULT

 None

-ip_version

Supported platforms Details

IxNetwork

DESCRIPTION

This is the ip type of the BFD session. This option is valid only when -mode create and -handle is provided or -mode is modify and -session_handle is provided.

Valid options are:

4

IPv4 version

6

IPv6 version

DEFAULT

4

-local_disc

Supported platforms Details

IxNetwork

DESCRIPTION

A unique, nonzero discriminator value generated by the interface, used to de-multiplex multiple BFD sessions between the same pair of systems.This option is valid only when -mode create and -handle is provided or -mode is modify and -session_handle is provided.

DEFAULT

1

-local_disc_step

Supported platforms Details

IxNetwork

DESCRIPTION

The local discriminator step. Used when creating multiple BFD sessions.This option is valid only when -mode create.

DEFAULT

1

-local_ip_addr

Supported platforms Details

IxNetwork

DESCRIPTION

Sets the source IP address for this BFD session. Only enabled if enable_auto_choose_source is 0.This option is valid only when -mode create and -handle is provided or -mode is modify and -session_handle is provided.

DEFAULT

 None

-local_ip_addr_step

Supported platforms Details

IxNetwork

DESCRIPTION

The source IP address step for the BFD session. Used when creating multiple BFD sessions.This option is valid only when -mode create.(DEFAULT = 0.0.0.1 | 0::1)

DEFAULT

0.0.0.1 | 0::1

-mode

Supported platforms Details

IxNetwork

DESCRIPTION

The action that will be taken for a specified handle or session_handle. Parameter -handle is mandatory for -mode create and parameter -session_handle is mandatory for -mode modify/enable/disable/delete.

Valid options are:

create

Creates BFD session(s). Parameter -handle needs to be provided. If -handle is a router handle, then the session will be configured on each router interface. If -handle is a router interface handle, then the session will be configured for that specific router interface. The router handle and the router interface handle are obtained from the returned keys of a previous call to emulation_bfd_config.

modify/enable/disable/delete

Performs the specified operation for a BFD session identified by -session_handle parameter.

DEFAULT

create

-remote_disc

Supported platforms Details

IxNetwork

DESCRIPTION

The discriminator received from the corresponding remote system. This field reflects back the received value of local_disc, or is zero if that value is unknown.This option is valid only when -mode create and -handle is provided or -mode is modify and -session_handle is provided.

DEFAULT

1

-remote_disc_step

Supported platforms Details

IxNetwork

DESCRIPTION

The step for the remote discriminator, when creating multiple sessions.This option is valid only when -mode create and -handle is provided.

DEFAULT

1

-remote_ip_addr

Supported platforms Details

IxNetwork

DESCRIPTION

The IP address of the remote router in the BFD session.This option is valid only when -mode create and -handle is provided or -mode is modify and -session_handle is provided. Required for -mode create.

DEFAULT

0.0.0.0

-remote_ip_addr_step

Supported platforms Details

IxNetwork

DESCRIPTION

The IP address step of the remote router in the BFD session, when creating multiple BFD sessions.This option is valid only when -mode create and -handle is provided.(DEFAULT = 0.0.0.1 | 0::1)

DEFAULT

0.0.0.1 | 0::1

-session_handle

Supported platforms Details

IxNetwork

DESCRIPTION

This is the BFD session handle that needs to be provided when modifying/enabling/disabling/deleting a BFD session. A list of session handles can be provided. This is mandatory when -mode is modify/enable/disable/delete.

DEFAULT

 None

-session_type

Supported platforms Details

IxNetwork

DESCRIPTION

The type of BFD session. This option is valid only when -mode create and -handle is provided or -mode is modify and -session_handle is provided.

Valid options are:

single_hop

the BFD session is between adjacent BFD routers.

multi_hop

the BFD session traverses more than one router.

DEFAULT

single_hop

RETURN VALUES

Supported Platform Details

All

status

$::SUCCESS | $::FAILURE

log

On status of failure, gives detailed information.

session_handles

On mode create, list of session handles
