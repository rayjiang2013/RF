NAME - ::<namespace>:: emulation_stp_msti_config

DESCRIPTION

IxNetwork, IxOS/IxNetwork-FT

This command is used to create/modify/delete/enable/disable a STP MSTI object. This object can be created on a MSTP bridge object. After this object is created it should be assigned to bridge - with mode modify of ixia

ARGUMENTS

[M] indicates if the argument is mandatory on the supported platform.

-bridge_handle

Supported platforms Details

IxNetwork

DESCRIPTION

Bridge handle on which to create MSTI objects. This option is mandatory for create mode. Option is valid only when -mode create is provided.

DEFAULT

 None

-count

Supported platforms Details

IxNetwork

DESCRIPTION

The number of MSTI objects to be created. This option is valid only when -mode create and -bridge_handle is provided.

DEFAULT

1

-handle

Supported platforms Details

IxNetwork

DESCRIPTION

Handle returned by this procedure for specified for modify purpose. This option is valid only when -mode is modify, delete, enable or disable.

DEFAULT

 None

-mode

Supported platforms Details

IxNetwork

DESCRIPTION

Action to be taken for: create, modify, delete, enable or disable. For mode create port_handle should be specified. For others handle should be provided.

DEFAULT

create

-msti_hops

Supported platforms Details

IxNetwork

DESCRIPTION

The number of MSTI hops remaining. An unsigned integer. Valid range: 1 to 255. This option is valid only when: -mode is create or -mode is modify and -handle is provided.

DEFAULT

20

-msti_id

Supported platforms Details

IxNetwork

DESCRIPTION

The identifier for this MSTI. An unsigned integer. The valid range is 1 to 4094. This option is valid only when: -mode is create or -mode is modify and -handle is provided.(DEFAULT = 1 for first object, 2 for second, etc.)

DEFAULT

1

-msti_id_step

Supported platforms Details

IxNetwork

DESCRIPTION

The amount used to increment msti_id. This option is valid only when -mode create and -bridge_handle is provided.

DEFAULT

0

-msti_internal_root_path_cost

Supported platforms Details

IxNetwork

DESCRIPTION

The MSTI Internal Root Path Cost. A 4-byte unsigned integer. This option is valid only when: -mode is create or -mode is modify and -handle is provided.

DEFAULT

0

-msti_mac

Supported platforms Details

IxNetwork

DESCRIPTION

The 6-byte MAC address for the MSTI root. This is part of the MSTI Regional Root Identifier. This option is valid only when -mode create and -bridge_handle is provided or -mode modify and -handle is provided.

DEFAULT

0000.0000.0001

-msti_mac_step

Supported platforms Details

IxNetwork

DESCRIPTION

The 6-byte MAC address step for the MSTI root. This is part of the MSTI Regional Root Identifier. This option is valid only when: -mode is create or -mode is modify and -handle is provided.

DEFAULT

0000.0000.0001

-msti_name

Supported platforms Details

IxNetwork

DESCRIPTION

The name assigned to this MST Instance (MSTI). This option specifies the MSTI name. If the character '%' is present in the MSTI name and the parameter -msti_wildcard_percent_enable is set to 1, then the '%' character will be replaced with the value specified through -msti_wildcard_percent_start. The value specified through -msti_wildcard_percent_start, will be incremented with the amount specified by -msti_wildcard_percent_step, if -count is greater than 1.(DEFAULT = for first object MSTI ID-1, for second MSTI ID-2, etc.)

DEFAULT

"MSTI ID-%"

-msti_port_priority

Supported platforms Details

IxNetwork

DESCRIPTION

MSTI Port Priority is part of the MSTI Port Identifier. Choose one of the following values from the list: 0 16 32 48 64 80 96 112 128 144 160 176 192 208 224 240.This option is valid only when: -mode is create or -mode is modify and -handle is provided.

DEFAULT

0

-msti_priority

Supported platforms Details

IxNetwork

DESCRIPTION

The MSTI Root Priority. This is part of the MSTI Regional Root Identifier. Since MAC address reduction is used, only multiples of 4096 are used. Choose one of the following values from the list: 0 4096 8192 12288 16384 20480 24576 28672 32768 36864 40960 45056 49152 53248 57344 61440This option is valid only when: -mode is create or -mode is modify and -handle is provided.

DEFAULT

32768

-msti_vlan_start

Supported platforms Details

IxNetwork

DESCRIPTION

The ID for the first VLAN in the VLAN range to which the MSTI is mapped. This option is valid only when: -mode is create or -mode is modify and -handle is provided.

DEFAULT

1

-msti_vlan_start_step

Supported platforms Details

IxNetwork

DESCRIPTION

The step for the first VLAN in the VLAN range to which the MSTI is mapped. This option is valid only when: -mode is create or -mode is modify and -handle is provided.

DEFAULT

1

-msti_vlan_stop

Supported platforms Details

IxNetwork

DESCRIPTION

The ID for the last VLAN in the VLAN range to which the MSTI is mapped. An unsigned integer. This option is valid only when: -mode is create or -mode is modify and -handle is provided.

DEFAULT

1

-msti_vlan_stop_step

Supported platforms Details

IxNetwork

DESCRIPTION

The step for the last VLAN in the VLAN range to which the MSTI is mapped. An unsigned integer should be provided. This option is valid only when: -mode is create or -mode is modify and -handle is provided.

DEFAULT

1

-msti_wildcard_percent_enable

Supported platforms Details

IxNetwork

DESCRIPTION

This parameter enables the replacement of % character found in -msti_name parameter, with the value specified through msti_wilcard_percent_start. For the first MSTI object created, the % character will be replaced with -msti_wilcard_percent_start. When -count is greater than 1, the -msti_wildcard_percent_start value will be incremented with -msti_wildcard_percent_step.

DEFAULT

1

-msti_wildcard_percent_start

Supported platforms Details

IxNetwork

DESCRIPTION

This parameter represents the value that will replace the % character used inside the string provided by -msti_name parameter.

DEFAULT

1

RETURN VALUES

Supported Platform Details

IxOS/IxNetwork-FT and IxNetwork

status

$::SUCCESS or $::FAILURE

log

When status is failure, contains more information

handle

On mode create, list of bridge handles
