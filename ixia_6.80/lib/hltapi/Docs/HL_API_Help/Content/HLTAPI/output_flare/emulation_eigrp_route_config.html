NAME - ::<namespace>:: emulation_eigrp_route_config

DESCRIPTION

IxNetwork, IxOS/IxNetwork-FT

This command is used to create/modify/delete/enable/disable a EIGRP session on a EIGRP router interface previously created with ::::emulation_eigrp_config. One EIGRP session can be added to all router interfaces or just to a specific router interface.

DESCRIPTION

(in kbps) The minimum amount of bandwidth available on this link. The valid range is 10000-4294967295.This is an optional argument. This option is valid only when -mode is create and -handle is provided as an EIGRP router or when -mode modify and -route_handle is provided as an EIGRP route.

-count

Supported platforms Details

IxNetwork

DESCRIPTION

The number of route ranges to create for an EIGRP router. The valid range is 1-1-16777215.This is an optional argument. This option is valid only when -mode is create and -handle is provided as an EIGRP router or when -mode modify and -route_handle is provided as an EIGRP route.

-delay

Supported platforms Details

IxNetwork

DESCRIPTION

(in 10s of Microseconds) The total of delays on the path to the route/network. The valid range is 0-4294967295.This is an optional argument. This option is valid only when -mode is create and -handle is provided as an EIGRP router or when -mode modify and -route_handle is provided as an EIGRP route.

DEFAULT

0

-dst_count

Supported platforms Details

IxNetwork

DESCRIPTION

Available for use only if enable_packing is enabled. The maximum number of destinations to be packed into a single Internal/External TLV. The valid range is 0-255. The default is 90. A value of 0 means that maximum possible packing will be used, which depends on the MTU of the link.This is an optional argument. If enable_packing is disabled this value is ignored. This option is valid only when -mode is create and -handle is provided as an EIGRP router or when -mode modify and -route_handle is provided as an EIGRP route.

DEFAULT

90

-enable_packing

Supported platforms Details

IxNetwork

DESCRIPTION

If 1/enabled, Packing is enabled. Multiple destinations will be packed into a single Internal/External TLV. If 0/disabled, only one destination will be packed into a single Internal/External TLV. This is an optional argument. This option is valid only when -mode is create and -handle is provided as an EIGRP router or when -mode modify and -route_handle is provided as an EIGRP route.

DEFAULT

1

-ext_flag

Supported platforms Details

IxNetwork

DESCRIPTION

Available only for external route ranges (-type external). This parameter represents the origin of the advertised route. This is an optional argument.If -type is internal, this parameter is ignored. This option is valid only when -mode is create and -handle is provided as an EIGRP router or when -mode modify and -route_handle is provided as an EIGRP route.

Valid options are:

external_route

the route was originated outside the EIGRP domain.

candidate_default

a route that could also serve as a default route.

DEFAULT

external_route

-ext_metric

Supported platforms Details

IxNetwork

DESCRIPTION

Available only for external route ranges (-type external). This parameter represents the EIGRP vector metric for the cost of the path to this route/network. The valid range is 1-4294967295.This is an optional argument. If -type is internal, this parameter is ignored. This option is valid only when -mode is create and -handle is provided as an EIGRP router or when -mode modify and -route_handle is provided as an EIGRP route.

DEFAULT

1

-ext_originating_as

Supported platforms Details

IxNetwork

DESCRIPTION

If -type is internal, this parameter is ignored. This option is valid only when -mode is create and -handle is provided as an EIGRP router or when -mode modify and -route_handle is provided as an EIGRP route.

DEFAULT

1

-ext_protocol

Supported platforms Details

IxNetwork

DESCRIPTION

Available only for external route ranges (-type external). This parameter represents the external protocol where the route was originated, if applicable.This is an optional argument. If -type is internal, this parameter is ignored. This option is valid only when -mode is create and -handle is provided as an EIGRP router or when -mode modify and -route_handle is provided as an EIGRP route.

Valid options are:

igrp

Interior Gateway Routing Protocol (the default)

eigrp

Enhanced Interior Gateway Routing Protocol

static

routes are statically-configured; no routing protocol is being used.

rip

Routing Information Protocol

hello

Hello message protocol

ospf

Open Shortest Path First

isis

Intermediate System to Intermediate System

egp

Exterior Gateway Protocol

bgp

Border Gateway Protocol

idrp

Inter-Domain Routing Protocol

connected

direct connection.

DEFAULT

igrp

-ext_route_tag

Supported platforms Details

IxNetwork

DESCRIPTION

An administrative tag applied to the route when it is redistributed between EIGRP and an external protocol, to prevent routing loops. Used as a route mapping filter. The valid range is 0-4294967295. This is an optional argument. If -type is internal, this parameter is ignored. This option is valid only when -mode is create and -handle is provided as an EIGRP router or when -mode modify and -route_handle is provided as an EIGRP route.

DEFAULT

0

-ext_source

Supported platforms Details

IxNetwork

DESCRIPTION

The IP address for the external source of the route information, in dotted decimal format.This is an optional argument. If -type is internal, this parameter is ignored. This option is valid only when -mode is create and -handle is provided as an EIGRP router or when -mode modify and -route_handle is provided as an EIGRP route.

DEFAULT

0.0.0.0 | 0::0

-prefix_start

DEFAULT

Supported platforms Details

IxNetwork

The step for the first IP network/destination address to be used in creating a set of route ranges for multiple routers. Valid only if -handle is provide with a list of router handles. The format is IP dotted decimal format.This is an optional argument. This option is valid only when -mode is create and -handle is provided as an EIGRP router.(DEFAULT = 1.0.0.0 | 0:0:1:0::0).

DEFAULT

1.0.0.0 | 0:0:1:0::0

Supported platforms Details

IxNetwork

The reliability factor. The valid range is 0-255.This is an optional argument. This option is valid only when -mode is create and -handle is provided as an EIGRP router or when -mode modify and -route_handle is provided as an EIGRP route.

255

IxNetwork

 None

Supported platforms Details

IxNetwork

 None
