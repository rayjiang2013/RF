ARGUMENTS

[M] indicates if the argument is mandatory on the supported platform.

-mode

Supported platforms Details

IxNetwork [M]

DESCRIPTION

Action to perform. This parameter is supported using the following:

Valid options are:

create

Create new PBB Bridges on -port_handle.

modify

Modify PBB Bridges specified with -handle parameter.

enable

Enable PBB Bridges specified with -handle parameter.

disable

Disable PBB Bridges specified with -handle parameter.

remove

Remove PBB Bridges specified with -handle parameter.

DEFAULT

 None

-port_handle

Supported platforms Details

IxNetwork [M]

DESCRIPTION

Parameter -port_handle specifies the port to be configured with PBB. This parameter is supported using the following APIs: IxN.

DEFAULT

 None

-bridge_id

Supported platforms Details

IxNetwork

DESCRIPTION

The bridge MAC address. This parameter is supported using the following APIs: IxN.

DEFAULT

 None

-bridge_id_step

Supported platforms Details

IxNetwork

DESCRIPTION

The bridge MAC address step. This parameter is supported using the following APIs: IxN.

DEFAULT

 None

-count

Supported platforms Details

IxNetwork

DESCRIPTION

Number of bridges to configure on port port_handle. This parameter is supported using the following APIs: IxN.

DEFAULT

 None

-enable_optional_tlv_validation

Supported platforms Details

IxNetwork

DESCRIPTION

This parameter is configured for all PBB Bridges. This parameter is supported using the following APIs: IxN. 0 - Enable.1 - Disable.

DEFAULT

0

-enable_out_of_sequence_detection

Supported platforms Details

IxNetwork

DESCRIPTION

If 1, enables the detection of out of sequence CCM messages. This parameter is supported using the following APIs: IxN. 0 - Enable.1 - Disable.

DEFAULT

1

-ether_type

Supported platforms Details

IxNetwork

DESCRIPTION

Selects the ether type for the bridge. This parameter is supported using the following APIs: IxN. 8902 - 0x8902.88E6 - 0x88E6.

DEFAULT

8902

-handle

Supported platforms Details

IxNetwork

DESCRIPTION

PBB handle. Used when -mode is modify|enable|disable|remove. This parameter is supported using the following APIs: IxN.

DEFAULT

 None

-interface_handle

Supported platforms Details

IxNetwork

DESCRIPTION

List of interface_handles to associate with PBB. Length must be equal to the value of -count parameter. Interface handles are obtained using ::::interface_config procedure. This parameter is supported using the following APIs: IxN.

DEFAULT

 None

-mac_address_init

Supported platforms Details

IxNetwork

DESCRIPTION

Protocol interface parmater. Starting MAC address. This parameter is supported using the following APIs: IxN.

DEFAULT

 None

-mac_address_step

Supported platforms Details

IxNetwork

DESCRIPTION

Protocol interface parmater. Mac address step. This parameter is supported using the following APIs: IxN.

DEFAULT

00:00:00:00:00:01

-receive_ccm

Supported platforms Details

IxNetwork

DESCRIPTION

If 1, the PBB protocol can receive PBB CCMs on this port. This parameter is supported using the following APIs: IxN. 0 - Disable.1 - Enable.

DEFAULT

1

-reset

Supported platforms Details

IxNetwork

DESCRIPTION

If 1 deletes all PBB bridges created on port -port_handle before creating new ones. This parameter is supported using the following APIs: IxN. 0 - Disable.1 - Enable.

DEFAULT

0

-send_ccm

Supported platforms Details

IxNetwork

DESCRIPTION

If 1, the PBB protocol can send PBB CCMs from this port. This parameter is supported using the following APIs: IxN. 0 - Disable.1 - Enable.

DEFAULT

1

-vlan_id

Supported platforms Details

DESCRIPTION

Protocol interface parmater. This parameter is supported using the following APIs: IxN.

DEFAULT

0

-vlan_id_step

Supported platforms Details

IxNetwork

DESCRIPTION

DEFAULT

1
