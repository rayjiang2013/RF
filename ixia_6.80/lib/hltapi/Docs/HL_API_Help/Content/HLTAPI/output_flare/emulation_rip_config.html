IxOS/IxNetwork-FT and IxNetwork

Coded versus functional specification.The router_id option will not be set properly on hardware.For IxNetwork the following new RIPng features are available:RIPng routers can have multiple RIPng router interfaces.By specifying the intf_... parameters there will be created a number equal to intf_count of directly connected interfaces per RIPng router.By specifying the gre_... parameters there will be created a number equal to gre_count of GRE tunnel per connected.At the end the number of RIPng router interfaces per RIPng router will be intf_count * gre_count (can not be greater than 100).Protocol interfaces created with ::<namespace>::emulation_rip_config can not be modified.Each RIPng router interface will be associated to:A directly connected interface, if the user specifies only the intf_... parametersA GRE tunnel, if the user specifies the intf_ and the gre_... parametersWhen -mode is modify and -handle contains only one RIPng router handle, the router interface parameters must have either one value, in which case the value will be applied to all interfaces of the associated RIPng router, either the same number of values as the number of interfaces that the associated RIPng router has and each router interface will take the corresponding value from the list.When -mode is modify and -handle contains a list of RIPng router handles, the interface parameters must have either one value, in which case all interfaces of all routers will be assigned with that value, either a list of values equal to the number of RIPng router handles and all interfaces of a router handle will take the corresponding value from the list of arguments.Example: -handle [list $ripR1 $ripR2 $ripR3] -interface_metric [list 3 4 5] All interfaces of $ripR1 will have a metric of 3; all interfaces of $ripR2 will have a metric of 4 and all interfaces of ripR3 will have a metric of 5.

IxOS/IxNetwork-FT and IxNetwork

Coded versus functional specification.The router_id option will not be set properly on hardware.For IxNetwork the following new RIPng features are available:RIPng routers can have multiple RIPng router interfaces.By specifying the intf_... parameters there will be created a number equal to intf_count of directly connected interfaces per RIPng router.By specifying the gre_... parameters there will be created a number equal to gre_count of GRE tunnel per connected.At the end the number of RIPng router interfaces per RIPng router will be intf_count * gre_count (can not be greater than 100).Protocol interfaces created with ::<namespace>::emulation_rip_config can not be modified.Each RIPng router interface will be associated to:A directly connected interface, if the user specifies only the intf_... parametersA GRE tunnel, if the user specifies the intf_ and the gre_... parametersWhen -mode is modify and -handle contains only one RIPng router handle, the router interface parameters must have either one value, in which case the value will be applied to all interfaces of the associated RIPng router, either the same number of values as the number of interfaces that the associated RIPng router has and each router interface will take the corresponding value from the list.When -mode is modify and -handle contains a list of RIPng router handles, the interface parameters must have either one value, in which case all interfaces of all routers will be assigned with that value, either a list of values equal to the number of RIPng router handles and all interfaces of a router handle will take the corresponding value from the list of arguments.Example: -handle [list $ripR1 $ripR2 $ripR3] -interface_metric [list 3 4 5] All interfaces of $ripR1 will have a metric of 3; all interfaces of $ripR2 will have a metric of 4 and all interfaces of ripR3 will have a metric of 5.
