NAME - ::<namespace>:: emulation_stp_lan_config

DESCRIPTION

IxNetwork, IxOS/IxNetwork-FT

This command is used to create/modify/delete/enable/disable an emulated LAN for STP protocol.

ARGUMENTS

[M] indicates if the argument is mandatory on the supported platform.

-count

Supported platforms Details

IxNetwork

DESCRIPTION

Number of mac/vlan to be emulated with current object.This option is valid only when -mode create is provided.

DEFAULT

1

-mac_address

Supported platforms Details

IxNetwork

DESCRIPTION

First mac address in the range. option is valid only when -mode create is provided or -mode modify and -handle is provided.

DEFAULT

0000.0000.0000

-mac_incr_enable

Supported platforms Details

IxNetwork

DESCRIPTION

If this option is enable then will be emulated -count of distinct mac generated incrementing by one. Option is valid only when -mode create is provided or -mode modify and -handle is provided.

DEFAULT

0

-mode

Supported platforms Details

IxNetwork

DESCRIPTION

Select action to take with current call. Choices are create, modify, delete, enable and disable. For mode create port_handle shoud be specified. For others handle should be provided.

DEFAULT

create

-port_handle

Supported platforms Details

IxNetwork

DESCRIPTION

Select port on which to create emulated LAN. option is valid only when -mode create is provided.

DEFAULT

 None

-vlan_enable

Supported platforms Details

IxNetwork

DESCRIPTION

Enable/disable VLAN for current LAN. This option is valid for create ormodify mode.

DEFAULT

0

-vlan_id

Supported platforms Details

IxNetwork

DESCRIPTION

First vlan to be generated in current range. option is valid only when -mode create and -bridge_handle is provided or -mode modify and -handle is provided.

DEFAULT

1

RETURN VALUES

Supported Platform Details

IxOS/IxNetwork-FT and IxNetwork

status

$::SUCCESS or $::FAILURE

log

When status is failure, contains more information

handle

The list of bridges created
