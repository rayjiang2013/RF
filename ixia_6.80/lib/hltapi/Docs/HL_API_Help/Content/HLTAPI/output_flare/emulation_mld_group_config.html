NAME - ::<namespace>:: emulation_mld_group_config

DESCRIPTION

ARGUMENTS

[M] indicates if the argument is mandatory on the supported platform.

-mode

Supported platforms Details

IxNetwork [M] , IxOS/IxNetwork-FT [M]

DESCRIPTION

This option defines the action to be taken.

Valid options are:

create

When this mode is used, parameters -session_handle and -group_pool_handle are mandatory. The command will create a group range on the selected mld host.

delete

Parameter -handle must be provided. In this case, -handle can represent one or multiple Group Ranges.

modify

Parameter -handle must be provided. In this case, -handle can represent one or multiple Group Ranges.

clear_all

When this mode is used, you must provide -session_handle.

DEFAULT

 None

IxNetwork-NGPF [M]

DESCRIPTION

This option defines the action to be taken.

Valid options are:

create

When this mode is used, you must provide -session_handle. The parameter -group_pool_handle is also mandatory.

delete

Parameter -handle must be provided. In this case, -handle can represent one or multiple Group Ranges.

modify

Parameter -handle must be provided. When using this mode, -handle can represent one or multiple Group Ranges.

clear_all

When this mode is used, you must provide -session_handle.

enable

When this mode is used, you must provide -handle.

disable

When this mode is used, you must provide -handle.

DEFAULT

 None

-return_detailed_handles

Supported platforms Details

IxNetwork-NGPF

DESCRIPTION

This argument determines if individual interface, session or router handles are returned by the current command. This applies only to the command on which it is specified. Setting this to 0 means that only NGPF-specific protocol stack handles will be returned. This will significantly decrease the size of command results and speed up script execution. The default is 0, meaning only protocol stack handles will be returned.

DEFAULT

0

-g_enable_packing

Supported platforms Details

IxNetwork

DESCRIPTION

If checked, multiple Sources can be included in each Record, and multiple Records can be included in each Frame, for more efficient transmission. This option is available only for MLDv2. This is valid only for IxTclNetwork API. If this parameter is not provided, the enable_packing that was set at host level will be used instead.

DEFAULT

 None

Supported platforms Details

IxNetwork

DESCRIPTION

Valid options are:

include

The specified source(s) is (are) of interest

exclude

The specified source(s) is (are) not of interest

DEFAULT

 None

-source_pool_handle

Supported platforms Details

IxNetwork, IxOS/IxNetwork-FT

DESCRIPTION

Associate source pool(s) with group(s). Specify one or more source pool handle(s) for (S,G) entries. None for (*,G) entries. The source pool(s) must be added beforehand through procedure emulation_multicast_source_config.

If the beforehand created source group (with emulation_multicast_source_config) has -ip_addr_step 0::1, for the Source Count attribute the -num_sources value will be taken in consideration.

If the beforehand created source group (with emulation_multicast_source_config) has -ip_addr_step different than 0::1, the Source Count will be 1 and a total of -ip_addr_step converted to integer- will be created.

DEFAULT

 None

IxNetwork-NGPF

DESCRIPTION

Associate source pool(s) with group(s). Specify one or more source pool handle(s) for (S,G) entries. None for (*,G) entries. The source pool(s) must be added beforehand through procedure emulation_multicast_source_config.

DEFAULT

 None
