NAME - ::<namespace>:: emulation_cfm_config

DESCRIPTION

IxNetwork, IxOS/IxNetwork-FT

This procedure will configure CFM bridges. This protocol is available only when using IxTclNetwork. The GUI equivalent is in the section Protocols/Carrier Ethernet where CFM/Y.1731/PBB-TE is enabled.

-mode

Supported platforms Details

IxNetwork [M]

DESCRIPTION

Action to perform. This parameter is supported using the following:

Valid options are:

create

Create new CFM Bridges on -port_handle.

modify

- Modify CFM Bridge specified with -handle parameter.

enable

- Enable CFM Bridges specified with -handle parameter.

disable

Disable CFM Bridges specified with -handle parameter.

remove

Remove CFM Bridges specified with -handle parameter.

DEFAULT

 None

-port_handle

Supported platforms Details

IxNetwork [M]

DESCRIPTION

Parameter -port_handle specifies the port to be configured with CFM. This parameter is supported using the following APIs: IxN.This parameter is used only when -mode is "create".

DEFAULT

 None

-ais_interval

Supported platforms Details

IxNetwork

DESCRIPTION

The interval between AIS messages sent from this CFM bridge.This parameter is supported using the following APIs: IxN.

Valid options are:

one_sec

The interval is one second.

one_min

The interval is one minute.

DEFAULT

one_sec

-allow_cfm_maid_formats

Supported platforms Details

IxNetwork

DESCRIPTION

If checked, allows to use CFM's MD Name types and Short MA Name types when the Operation Mode is Y.1731. This parameter can be enabled and becomes active only when the operation mode is Y.1731. This parameter is disabled when operation mode is either CFM (IEEE 802.1ag) or PBB-TE.

DEFAULT

0

-handle

Supported platforms Details

IxNetwork

DESCRIPTION

CFM Bridge handle. Used when -mode is modify|enable|disable|remove. When mode is "modify" this parameter supports only one CFM Bridge handle.When mode is enable|disable|remove this parameter supports a list of CFM Bridges. This parameter is supported using the following APIs: IxN.

DEFAULT

 None

-interface_handle

Supported platforms Details

IxNetwork

DESCRIPTION

List of interface_handles to associate with CFM Bridges. Length must be equal to the value of -count parameter. Interface handles are obtained using ::::interface_config procedure. This parameter is supported using the following APIs: IxN.This parameter is used only when -mode is create|modify.When -mode is "modify" this parameter will map the protocol interface specified with -interface_handle to the CFM Bridge specified with -handle parameter.

DEFAULT

 None

-mac_address_init

Supported platforms Details

DESCRIPTION

DEFAULT

DESCRIPTION

DEFAULT

-operation_mode

Supported platforms Details

IxNetwork

DESCRIPTION

Selects the type of CFM to enable. This parameter is supported using the following APIs: IxN.

Valid options are:

cfm

Uses the IEEE 802.1ag version.

y1731

Uses the ITU-T Y.1731 version.

DEFAULT

cfm

-override_existence_check

Supported platforms Details

IxNetwork

DESCRIPTION

If this option is enabled, the interface existence check is skipped but the list of interfaces is still created and maintained in order to keep track of existing interfaces if required. Using this option will speed up the the creation of the interfaces.

DEFAULT

0

-override_tracking

Supported platforms Details

IxNetwork

DESCRIPTION

If this option is enabled, the list of interfaces won't be created and maintained anymore, thus, speeding up the interfaces' creation even more. Also, it will enable -override_existence_check in case it was not already enabled because checking for interface existence becomes impossible if the the list of interfaces does not exist anymore.

DEFAULT

0

-receive_ccm

Supported platforms Details

IxNetwork

DESCRIPTION

Valid options are:

0

1

override_existence_check

Supported platforms Details

IxNetwork

DESCRIPTION

If this option is enabled, the interface existence check is skipped but the list of interfaces is still created and maintained in order to keep track of existing interfaces if required. Using this option will speed up the the creation of the interfaces.

DEFAULT

0

override_tracking

Supported platforms Details

IxNetwork

DESCRIPTION

If this option is enabled, the list of interfaces won't be created and maintained anymore, thus, speeding up the interfaces' creation even more. Also, it will enable -override_existence_check in case it was not already enabled because checking for interface existence becomes impossible if the the list of interfaces does not exist anymore.

DEFAULT

0
