-mode

Supported platforms Details

IxNetwork-NGPF

DESCRIPTION

The mode parameter is only needed when specifying values for custom_handle or increment_handle. When a custom_handle is specified the only valid mode values are - create - modify When a increment_handle is specified all choices are valid.

Valid options are:

create

creates a new CustomIncrement with the specified values

modify

updates the root CustomIncrement with the specified values

destroy

-custom_distribution_algorithm

Supported platforms Details

IxNetwork-NGPF

DESCRIPTION

The algorithm that will be used to distribute the values specified in custom_distribution_values.

Valid options are:

auto_even

The values specified by custom_distribution_values will be distributed evenly.

auto_geometric

The values specified by custom_distribution_values will be distributed geometrically.

percentage

The values specified by custom_distribution_values will be distributed according to the percentages specified in custom_distribution_percentages.

weighted

The values specified by custom_distribution_values will be distributed accorgind to the weights specified in custom_distribution_percentages.

DEFAULT

percentage

DEPENDENCIES

Valid in combination with parameter(s)
'pattern' | value= 'custom_distributed' |

-custom_distribution_mode

Supported platforms Details

IxNetwork-NGPF

DESCRIPTION

The mode in which the distribution algorithm specified by custom_distribution_algorithm will be applied.

Valid options are:

topology

The values specified in custom_distribution_values will be distributed across all the configured devices according to the distribution algorithm specified by custom_distribution_algorithm.

device

The values specified in custom_distribution_values will be distributed across each set of devices configured for a parent device according to the distribution algorithm specified by custom_distribution_algorithm.

port

The values specified in custom_distribution_values will be distributed across each set of devices configured on a port according to the distribution algorithm specified by custom_distribution_algorithm.

DEFAULT

topology

DEPENDENCIES

Valid in combination with parameter(s)
'pattern' | value= 'custom_distributed' |

DESCRIPTION

The full path of a local file that will be used to import the list of values for the value_list pattern. The file should be in either CSV, CISCO or Juniper format. This argument's value will be used only if th values_list argument was not manually specified.

DEFAULT

 None

DEPENDENCIES

Valid in combination with parameter(s)
'pattern' | value= 'value_list' |

-values_file_type

Supported platforms Details

IxNetwork-NGPF

DESCRIPTION

The type of file that will be used to import the list of values for the value_list pattern. This argument is used only when values_file is specified and used as well and it has a default value of csv.

Valid options are:

cisco

Indicates that the file specified by values_file has a CISCO format.

csv

Indicates that the file specified by values_file has a CSV format.

juniper

Indicates that the file specified by values_file has a JUNIPER format.

unsupported

Indicates that the file specified by values_file has an unsupported format.

DEFAULT

csv

DEPENDENCIES

Valid in combination with parameter(s)
'pattern' | value= 'value_list' |

-values_file_column_index

Supported platforms Details

IxNetwork-NGPF

DESCRIPTION

The 0-based index of the column that will be used to import the values. This argument is used only when values_file is specified and used as well and it has a default value of 0.

DEFAULT

0

Valid in combination with parameter(s)
'pattern' | value= 'value_list' |

DESCRIPTION
