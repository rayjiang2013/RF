-action

Supported platforms Details

IxNetwork, IxOS/IxNetwork-FT

DESCRIPTION

Supported with IxTclHal. Controls the action of the buffer when it reaches the full status. Not supported with IxTclNetwork and warning will be printed on stdout if this parameter is used.DEFAULT: wrap

DEFAULT

wrap

-after_trigger_filter

Supported platforms Details

IxNetwork, IxOS/IxNetwork-FT

DESCRIPTION

Supported with IxTclHal and IxTclNetwork. Controls the capture of data after triggering when operating

Valid options are:

all

capture all data after trigger.

filter

(default) capture filtered data after trigger using filter settings

condition_filter

capture filtered data after trigger as per the filter settings

DEFAULT

 None

-before_trigger_filter

Supported platforms Details

IxNetwork, IxOS/IxNetwork-FT

DESCRIPTION

Supported with IxTclHal and IxTclNetwork. Controls the capture of data prior to triggering when operating in triggered mode (capture_mode = trigger).

Valid options are:

all

capture all data before trigger.

none

(default) capture none of the data before trigger.

filter

capture filtered data before trigger as per the filter settings

DEFAULT

 None

-capture_mode

Supported platforms Details

IxNetwork, IxOS/IxNetwork-FT

DESCRIPTION

Supported with IxTclHal and IxTclNetwork. Controls whether data capture is performed in a continuous or triggered mode.

Valid options are:

continuous

capture data in the buffer continuously, regardless of trigger settings. Data may be filtered; see continuous_filter.

trigger

(default) capture data only after triggered.

DEFAULT

 None

-continuous_filter

Supported platforms Details

IxNetwork, IxOS/IxNetwork-FT

DESCRIPTION

Supported with IxTclHal and IxTclNetwork. Controls whether data captured in continous mode (capture_mode = continuous) is filtered or not.

Valid options are:

all

(default) capture all data, regardless of filter settings.

filter

capture only filtered data, as per filter settings.

DEFAULT

 None

-data_plane_capture_enable

Supported platforms Details

IxNetwork

DESCRIPTION

Supported with IxTclNetwork. If 1, enables the capture of data plane traffic.

DEFAULT

1

-control_plane_capture_enable

Supported platforms Details

IxNetwork

DESCRIPTION

Supported with IxTclNetwork. If 1, enables the capture of control plane traffic.

DEFAULT

1

-control_plane_filter_pcap

Supported platforms Details

IxNetwork, IxOS/IxNetwork-FT

DESCRIPTION

Supported with IxTclHal and IxTclNetwork. This is the filter string for control plane traffic capture (a valid pcap filter syntax). Valid only when control_plane_capture_enable is 1.

DEFAULT

 None

-control_plane_trigger_pcap

Supported platforms Details

IxNetwork

DESCRIPTION

Supported with IxTclNetwork. This is the trigger string for control plane traffic capture (a valid pcap filter syntax). Valid only when control_plane_capture_enable is 1.

DEFAULT

 None

-small_packet_capture

Supported platforms Details

IxNetwork, IxOS/IxNetwork-FT

DESCRIPTION

Supported with IxTclHal. Applies to OC12 cards only. Capture of packets of 48 bytes or less at full wire rates can be problematic and is usually treated as an error. This setting allows packets of 48 bytes or less in length to be captured. The data captured, however, may be corrupt. Not supported with IxTclNetwork and warning will be printed on stdout if this parameter is used.

DEFAULT

0

-slice_size

Supported platforms Details

IxNetwork, IxOS/IxNetwork-FT

DESCRIPTION

Supported with IxTclHal and IxTclNetwork. The maximum number of octets of each frame that will be saved in this capture buffer. For example, if a 1500 octet frame is received by the probe and this option is set to 500, then only 500 octets of the frame will be stored in the associated capture buffer. If this option is set to 0, the capture buffer will save as many octets as is possible. If the slice_size is set larger than the maximim hardware supported slice size, the maximum will be used.

DEFAULT

8191

-trigger_position

Supported platforms Details

IxNetwork, IxOS/IxNetwork-FT

DESCRIPTION

Supported with IxTclHal and IxTclNetwork. Controls the dividing line within the capture buffer between before trigger data and post trigger data. This control is only useful in triggered mode (capture_mode = trigger) and before trigger capture enabled (before_trigger_filter= all or filter). Trigger_position is expressed as a percentage of the total buffer size. The beginning of the buffer with this percentage is used in a wrap-around mode for before trigger data and the remainder is filled up with triggered data.

DEFAULT

1.0

-no_write

Supported platforms Details

IxNetwork, IxOS/IxNetwork-FT

DESCRIPTION

Supported with IxTclHal and IxTclNetwork. Whether to enable the procedure to commit change to the hardware

DEFAULT

 None

RETURN VALUES

Supported Platform Details

IxOS/IxNetwork-FT and IxNetwork

status

$::SUCCESS | $::FAILURE

log

If status is failure, detailed information provided.

port_handle

The port_handle received as a paramater

SAMPLE OUTPUT

NOTES

all

When calling ::<namespace>::packet_config_buffers a L1 reset will be triggered which will cause any protocol/traffic on that port to stop.
