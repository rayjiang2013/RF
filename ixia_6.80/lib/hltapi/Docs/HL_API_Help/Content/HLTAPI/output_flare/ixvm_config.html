NAME - ::<namespace>:: ixvm_config

DESCRIPTION

IxNetwork-NGPF

This command can be used to create and delete IxVM virtual cards on an IxVM virtual chassis. Please note that no operation can be performed if the cards is owned by any user. Ownership needs to be cleared before making any configuration changes.

ARGUMENTS

[M] indicates if the argument is mandatory on the supported platform.

-mode

Supported platforms Details

IxNetwork-NGPF [M]

DESCRIPTION

Not defined

Valid options are:

create

Create virtual card(s) of the specified virtual chassis.

modify

Modify virtual card(s) properties (including adding/removing virtual ports).

delete

Delete virtual card(s) or delete virtual ports from a virtual card if port_no argument is specified.

delete_all

Delete all virtual cards from the specified virtual chassis.

DEFAULT

 None

-port_action

Supported platforms Details

IxNetwork-NGPF

DESCRIPTION

Valid only for mode modify. When specified will allow adding/deleting port(s) to existing cards. Without specifying this parameter only existing port(s) can be modified. Please note that in this mode the existing port setting will not be modified (mtu, promiscuous. arguments supplied will only refer to the new ports being added).

Valid options are:

add

This action will allow adding new ports to existing cards. Please note than a maximum of 9 ports can be added to a card (if the card has 10 virtual interfaces - 9 interfaces + 1 management). This mode is similar with mode create, but instead of adding a new card will append to existing cards.

delete

This is similar with mode delete when port_no is present. Will delete the ports specified in the port_no from the card specified by management_ip or card_no.

DEFAULT

 None

DEPENDENCIES

Valid in combination with parameter(s)
'mode' | value= 'modify' |

-virtual_chassis

Supported platforms Details

IxNetwork-NGPF [M]

DESCRIPTION

The ip or hostname of the virtual chassis. If a DNS name is provided, please make sure the name can be resolved using the dns provider from the ixnetwork_tcl_server machine.

DEFAULT

 None

-card_no

Supported platforms Details

IxNetwork-NGPF

DESCRIPTION

The number of the card affected by the chosen action. Valid only for mode modify or delete. If management_ip is provided the card_no is ignored.

DEFAULT

 None

DEPENDENCIES

Valid in combination with parameter(s)
'mode' | value= 'modify' |
'mode' | value= 'delete' |

-port_no

Supported platforms Details

IxNetwork-NGPF

DESCRIPTION

The number of the port affected by the chosen action. Valid only for mode modify or delete_port.

DEFAULT

 None

DEPENDENCIES

Valid in combination with parameter(s)
'mode' | value= 'modify' |
'mode' | value= 'delete' |

-management_ip

Supported platforms Details

IxNetwork-NGPF

DESCRIPTION

The management IPv4 address of the virtual card.

DEFAULT

 None

DEPENDENCIES

Valid in combination with parameter(s)
'mode' | value= 'create' |
'mode' | value= 'modify' |
'mode' | value= 'delete' |

-keep_alive

Supported platforms Details

IxNetwork-NGPF

DESCRIPTION

The interval in seconds in which the keep alive is being sent to the virtual chassis.

DEFAULT

300

DEPENDENCIES

Valid in combination with parameter(s)
'mode' | value= 'create' |
'mode' | value= 'modify' |

-virtual_interface_list

Supported platforms Details

IxNetwork-NGPF

DESCRIPTION

The list of interface names that the virtual ports will use.
This argument is only valid when mode is create or modify. For mode modify this argument is only valid when port_action is add.

Example: providing a list of eth1, eth2, eth3 will result in 3 virtual ports using eth1, eth2 and respectively eth3 virtual network interface names.

DEFAULT

 None

DEPENDENCIES

Valid in combination with parameter(s)
'mode' | value= 'create' |
'mode' | value= 'modify' |

-virtual_interface_count

Supported platforms Details

IxNetwork-NGPF

DESCRIPTION

The number of ports to create. Using this argument the interface names are automatically calculated.
Can be used instead of virtual_interface_list.
This argument is only valid when mode is create or modify. For mode modify this argument is only valid when port_action is add.
When both virtual_interface_list and virtual_interface_count are present, virtual_interface_list will take precedence over virtual_interface_count and virtual_interface_count will be ignored.
Example: virtual_interface_count 5 will result in eth1 eth2 eth3 eth4 and eth5 as the 5 created virtual interfaces

DEFAULT

 None

DEPENDENCIES

Valid in combination with parameter(s)
'mode' | value= 'create' |
'mode' | value= 'modify' |

-mtu

Supported platforms Details

IxNetwork-NGPF

DESCRIPTION

The MTU of the virtual port created. When a single value is provided, the value will be applied to all ports given by the virtual_interface_list. When providing a list, the list length needs to match the length of the one provided for virtual_interface_list.

DEFAULT

1500

DEPENDENCIES

Valid in combination with parameter(s)
'mode' | value= 'create' |
'mode' | value= 'modify' |

-promiscuous_mode

Supported platforms Details

IxNetwork-NGPF

DESCRIPTION

This argument specifies if the virtual port will use promiscuous mode. For more info on this setting please consult IxVM documentation. When providing a single value, the value will be applied to all ports given by the virtual_interface_list. When providing a list, the list length needs to match the length of the one provided for virtual_interface_list.

DEFAULT

0

DEPENDENCIES

Valid in combination with parameter(s)
'mode' | value= 'create' |
'mode' | value= 'modify' |

-break_locks

Supported platforms Details

IxNetwork-NGPF

DESCRIPTION

This argument, if specified, will force clear ownership if the virtual card is in use (has ownership taken by any user). This will allow modify/delete operations to be performed forcefully. Please note that forcefully clearing port ownership will disconnect any existing user from the port causing current running script failure. Valid only when mode is modify, delete. delete_all. For mode create, the ownership cannot be cleared using break_locks since the ports do not exist in order to clear ownership. If the ports are in use by other IxOS Virtual chassis you need to first delete/modify them from that chassis (and here you can use the break_locks and then add them to the current chassis).

DEFAULT

 None

DEPENDENCIES

Valid in combination with parameter(s)
'mode' | value= 'delete' |
'mode' | value= 'delete_all' |
'mode' | value= 'modify' |

-rebuild_from_discovery

Supported platforms Details

IxNetwork-NGPF

DESCRIPTION

This argument, if specified, will rebuild entire topology using discovered appliances. This argument is only valid for mode create. When this argument is used only the following arguments are valid: virtual_chassis, rediscover and promiscuous_mode (only one value can be specified in this mode and this value will apply to all ports from all discovered appliances). Any other arguments will be ignored.

DEFAULT

 None

DEPENDENCIES

Valid in combination with parameter(s)
'mode' | value= 'create' |

-rediscover

Supported platforms Details

IxNetwork-NGPF

DESCRIPTION

If this argument is specified, a rediscovery will be done prior to rebuilding from discovered appliances. Valid only when mode is rebuild_from_discovery is used.

DEFAULT

 None

DEPENDENCIES

Valid in combination with parameter(s)
'moderebuild_from_discovery' | value= '1' |
'mode' | value= 'create' |

RETURN VALUES

Supported Platform Details

IxNetwork-NGPF

status

$::SUCCESS | $::FAILURE

card_no

The ID of the card created from the virtual chassis.

log

If status is failure, detailed information provided.
