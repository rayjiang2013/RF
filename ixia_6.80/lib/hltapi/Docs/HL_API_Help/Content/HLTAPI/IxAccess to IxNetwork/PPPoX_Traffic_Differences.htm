::ixia::pppox_stats

Parameter Description

port_handle

IxNetwork gatheres statistics per port (specified with port_handle).
IxAccess gatheres statistics per pppox_handle.

handle

This parameter was kept for backwards compatibility.

If this parameter is used in conjunction with IxNetwork, the statistics are gathered for the port on which the pppox_handle was configured.

Example:

::ixia::pppox_stats -handle $pppox_h_1

::ixia::pppox_stats -handle $pppox_h_2

Where

  • pppox_h_1 is on $port_1
  • pppox_h_2 is also on $port_1

Then both pppox_stats calls will return the same statistics because the statistics are actually taken from $port_1 (not per pppox_handle).

mode session

This mode is not supported with IxNetwork. Per pppox session statistics are not supported yet.

Statistic keys

IxNetwork aggregate keys are returned in the same format as IxAccess aggregate keys.

In addition, a new set of keys is returned similar to the IxAccess format but with the port_handle as primary key. (for example [<port_handle.>].aggregate.auth_total_rx). This is useful when a lis t of values is passed with the port_handle parameter.

The IxAccess compatible format (without port_handle) contains the statistics for the first port in the list.
